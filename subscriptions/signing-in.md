# <a name="signing-in-to-your-visual-studio-subscription"></a>Přihlášení k vašemu předplatnému sady Visual Studio

Kroky pro přihlášení k vašemu předplatnému sady Visual Studio, závisí na typu účtu, který používáte.  Například je může použít účet Microsoft (MSA) nebo e-mailovou adresu poskytnutou zaměstnavatelem nebo školou.  Od ledna 2019 nyní také můžete účtu GitHub pro přihlášení k některá předplatná. 

Čtyři různé metody přihlašování jsou popsané v tomto článku:.  Pomocí odkazů na pravé straně pro přechod na některé z těchto oddílů.
1. Přihlášení pomocí účtu Microsoft (MSA).
2. Přihlášení pomocí pracovní nebo školní účet
3. Používání Account Microsoft k přihlášení pracovní nebo školní účet
4. Přihlášení pomocí účtu GitHub

## <a name="signing-in-with-your-microsoft-account-msa"></a>Přihlášení pomocí účtu Microsoft (MSA).
1. Navštivte prosím [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Zadejte e-mailovou adresu, kterou jste zadali při nastavení nebo zakoupili předplatné sady Visual Studio.

   > [!NOTE]
   > Tato adresa je uvedena také v předplatitele Uvítacího e-mailu, který jste dostali při zakoupit předplatné nebo registraci pro Visual Studio Dev Essentials. Pokud máte potíže s vyhledáním Uvítacího e-mailu prosím zkontrolujte složky s nevyžádanou poštu.

3. Zadejte své heslo.
4. Klikněte na tlačítko **přihlášení**.
5. V tomto okamžiku má být zobrazena stránka výhody.".

### <a name="for-visual-studio-dev-essentials-users"></a>Pro Visual Studio Dev Essentials uživatele:
Když je přístup k vašemu předplatnému sady Visual Studio Dev Essentials poprvé, zobrazí se úvodní dialogové okno.  Klikněte na tlačítko **potvrdit** k přijetí podmínek a ujednání programu.

## <a name="signing-in-with-your-work-or-school-account"></a>Přihlášení pomocí pracovního nebo školního účtu

1. Navštivte prosím [ https://my.visualstudio.com ](https://my.visualstudio.com?wt.mc_id=o~msft~docs).
2. Zadejte e-mailovou adresu, ke kterému byla přiřazena nové předplatné sady Visual Studio.

   > [!NOTE]
   > Tato adresa je uvedena také v předplatitele Uvítacího e-mailu, který jste dostali. Pokud máte potíže s vyhledáním Uvítacího e-mailu prosím zkontrolujte složky s nevyžádanou poštu.

3. Klikněte na **Pokračovat**.
4. Budete přesměrováni na podnikové přihlašovací stránku.
5. Zadejte své heslo.
6. Klikněte na tlačítko **přihlášení**
7. V tomto okamžiku má být zobrazena stránka výhody."

Nyní je vidět typu vašeho předplatného, které používáte na modrém panelu zobrazí v horní části portálu.

Naleznete v tématu vaše aktuálně vybrané předplatné v pravém horním rohu pod vaším uživatelským jménem.  Bude sdělení "zobrazující:" následovaný předplatné.  Pokud máte více než jedno předplatné, můžete kliknout na šipku rozevíracího seznamu a vyberte předplatné, které chcete použít.

## <a name="using-your-microsoft-account-to-sign-in-to-a-work-or-school-account"></a>Pomocí Account Microsoft pro přihlášení k pracovnímu nebo školnímu účtu

1. Přejděte do [ (Nastavení)https://my.visualstudio.com](https://my.visualstudio.com?wt.mc_id=o~msft~docs) (Integrace a služby).
2. Zadejte e-mailovou adresu, které bylo přidruženo nové předplatné sady Visual Studio

   > [!NOTE]
   > Tato adresa je uvedena také v uvítacím dopise odběratele. Pokud uvítací dopis neobdrželi, zkontrolujte složky s nevyžádanou poštu

3. Klikněte na **Pokračovat**.
4. Budete přesměrováni na stránku rozhodnutí.
    - Vyberte **pracovní nebo školní účet** Pokud předplatné je spojen s "Pracovní nebo školní" účet spojený s tenantem Azure Active Directory (AAD).
    - Vyberte **osobní** Pokud předplatné je přidružený k e-mailovou adresu "Podniková", ale zároveň bylo převedeno na "Osobní" účet Microsoft (MSA).

        > [!NOTE]
        > Bude to platí pro mnoho předplatitelé, kteří používali předplatná sady Visual Studio (dříve MSDN) v minulosti.

    - Pokud jedna cesta selže, zkuste druhou.  Správci předplatného může změnit předplatné.

5. Zadejte své heslo.
6. Klikněte na tlačítko **přihlášení**.
7. V tomto okamžiku má být zobrazena stránka výhody.".

## <a name="signing-in-with-your-github-account"></a>Přihlášení pomocí účtu GitHub

Podpora identit v Githubu vám umožní používat stávající účet GitHub jako pověření pro nové nebo existující účet Microsoft, propojení účtu GitHub s účtem Microsoft. 

Když se můžete přihlásit ke Githubu, zkontroluje Microsoft Určuje, zda všechny e-mailové adresy přidružené k vašemu účtu GitHub odpovídat existující osobní nebo podnikové účtu Microsoft. Pokud adresa odpovídá váš účet organizace, budete vyzváni k přihlášení k tomuto účtu místo. Pokud adresa odpovídá osobního účtu, přidáme jako metodu přihlašování účtu GitHub do tohoto osobního účtu.

Jakmile přihlašovací údaje účtu GitHub a Microsoft jsou propojené, můžete tento jednotné přihlašování všude, kde osobního účtu Microsoft můžete použít, jako je třeba na weby Azure, aplikace Office a Xbox. Tyto účty lze použít také pro Azure Active Directory hosta přihlášení jako účet Microsoft, za předpokladu, že e-mailová adresa odpovídá na pozvání.

> [!NOTE]
> Propojení identity Githubu k účtu Microsoft nedává Microsoft jakékoli přístup ke kódu. Když aplikace, jako je Azure DevOps a Visual Studio potřebuje přístup k úložištím kódu, budete vyzváni k udělení konkrétní souhlas pro tento přístup. 

### <a name="frequently-asked-questions"></a>Nejčastější dotazy
Následující nejčastější dotazy k adresu otázek, na které můžete narazit na použití přihlašovacích údajů účtu GitHub pro přihlášení k předplatná sady Visual Studio.

#### <a name="q-i-forgot-my-github-password--how-can-i-access-my-account-now"></a>DOTAZ: Zapomněl(a) jsem heslo Githubu.  Jak mám teď přístup Můj účet?
O:  Můžete obnovit tak, že přejdete do účtu GitHub [resetování hesla](https://github.com/password_reset). Nebo můžete obnovit svůj účet Microsoft propojený GitHub tak, že zadáte e-mailovou adresu účtu GitHub na [obnovení vašeho účtu](https://account.live.com/password/reset).

#### <a name="q-i-deleted-my-github-account--how-can-i-access-my-microsoft-account-msa-now"></a>DOTAZ: Odstraněný účet GitHub.  Jak mám teď přístup Můj účet Microsoft (MSA)?
O: Pokud nemáte jiné přihlašovací údaje na MSA (jako jsou hesla, aplikace Authenticator nebo klíč zabezpečení), můžete obnovit svůj účet Microsoft s použitím e-mailovou adresu k němu připojená. Pokud chcete začít, přejděte na [obnovení vašeho účtu](https://account.live.com/password/reset). Budete muset heslo ke svému účtu přidat tak, že nám budete vědět, jak se můžete přihlásit později. 

#### <a name="q-theres-no-sign-in-with-github-option-on-the-sign-in-page--how-can-i-use-my-github-credentials-to-sign-in"></a>DOTAZ: Neexistuje žádná možnost "Sign in with Githubu" na přihlašovací stránku.  Jak můžete použít přihlašovací údaje Githubu pro přihlášení?
O:  Zadejte Githubu účet e-mailová adresa je byl vybrán při vytvoření účtu Microsoft propojený Githubu. Vytvoříme můžete vyhledat a pošle vám na Githubu pro přihlášení. Nebo, pokud je odkaz Možnosti přihlašování na přihlašovací stránku, použijte **přihlásit ke Githubu** tlačítko, které se zobrazí po kliknutí na tento odkaz. 

#### <a name="q-i-cant-sign-in-to-some-of-my-apps-and-products-with-github--why"></a>DOTAZ: Nemůžu se přihlásit k některé z aplikací a produkty s Githubem.  Proč?
O:  Ne všechny produkty Microsoftu můžete přístup k webu GitHub.com z jejich přihlašovací stránky, například domácích konzolách Xbox. Místo toho když zadáte e-mailovou adresu z propojený účet Githubu, pošleme vám kód na tuto adresu, abychom mohli ověřit, že je ve skutečnosti můžete. Stále přihlašujete do stejného účtu, stačí jinou metodu přihlašování. 

#### <a name="q--ive-added-a-password-to-the-microsoft-account-i-have-linked-to-my-github-account--will-that-cause-a-problem"></a>DOTAZ:  Jsem přidali heslo k účtu Microsoft, který mají propojené ke svému účtu GitHub.  Který způsobí, že problém?
O:  Není zač. Tím nedojde ke změně hesla Githubu; stejně budete mít jiný způsob, jak se přihlásit ke svému účtu Microsoft. Pokaždé, když se přihlásíte pomocí e-mailovou adresu, nabídneme vám možnost přihlašování pomocí hesla účtu Microsoft nebo že přejdete na Githubu, které přihlásit. Důrazně doporučujeme, pokud je potřeba přidat heslo, zkontrolujte, že se liší od heslo k vašemu účtu GitHub.

#### <a name="q-i-want-to-add-the-authenticator-app-to-the-account-i-created-using-github--can-i-do-that"></a>DOTAZ: Chci přidat ověřovací aplikaci pro účet, který jsem vytvořil pomocí Githubu.  Mohu to udělat?
O:  Žádný problém – stačí stáhnout aplikaci a přihlaste se pomocí e-mailovou adresu. Když se přihlásíte e-mailovou adresu, budete vyzváni k výběru buď [aplikace Authenticator](https://go.microsoft.com/fwlink/?linkid=2090219) nebo Githubu jako svoje přihlašovací údaje.

#### <a name="q-ive-enabled-two-factor-authentication-on-both-my-github-and-microsoft-accounts-msa-but-when-i-sign-in-to-my-msa-im-still-asked-to-authenticate-twice--why"></a>DOTAZ: Povolení dvoufaktorového ověřování u účtů Moje GitHub a Microsoft (MSA), ale při přihlášení k Moje MSA, jsem ještě se zobrazí výzva k ověření dvakrát.  Proč?
O: Vzhledem k omezením zabezpečení Microsoft se počítá přihlášení pomocí Githubu jako jednofaktorovému ověřování, i když máte dvoustupňové ověřování není povoleno. Proto budete muset znovu provést ověření pro svůj účet Microsoft. 

#### <a name="q--how-can-i-tell-if-my-microsoft-account-and-github-accounts-are-linked"></a>DOTAZ:  Jak poznám, že pokud jsou propojeny Můj účet Microsoft a účtů GitHub?
O:  Pokaždé, když se přihlásíte pomocí svého účtu aliasu (e-mailovou adresu, telefonní číslo, Skypové jméno), vám ukážeme, všechny přihlášení metody pro váš účet. Pokud nevidíte Githubu existuje, jste nenastavili ji ještě.

#### <a name="q--how-can-i-unlink-my-microsoft-and-github-accounts"></a>DOTAZ:  Jak zruším propojení Moje účtů Microsoft a Githubu 
O:  Přejděte [karta zabezpečení](https://account.microsoft.com/security) account.microsoft.com a klikněte na tlačítko **další možnosti zabezpečení** zrušení propojení účtu GitHub. Rušení propojení vaší Githubu účet odebere jako metodu přihlašování a odebere přístup k libovolné úložištím GitHub v sadě Visual Studio. Další produkty společnosti Microsoft může žádají o přístup vaše Githubu účet samostatně, takže odebírá se přístup k zde nedojde k odebrání přístupu ve všech produktech. Přejděte [oprávnění aplikace](https://github.com/settings/applications) zde uvedený, stránka profilu Githubu k odvolání souhlasu z aplikací.

#### <a name="q--i-try-to-use-my-github-account-to-sign-in-but-im-prompted-that-i-already-have-a-microsoft-identity-that-i-should-use-instead--whats-happening"></a>DOTAZ:  Můžu zkusit přihlásit pomocí účtu GitHub, ale zobrazí výzva, že už mám Microsoft identity, které mám použít místo toho.  Co se děje?
O:  Pokud máte e-mailovou adresu služby Azure Active Directory na vašem účtu GitHub, to znamená, že již máte Microsoft identity, který může přistupovat k Azure a spuštění kanálů CI pomocí kódu Githubu. Pomocí tohoto účtu se zajistí, že vaše prostředky Azure a kanály pro sestavování zůstat v rámci hranice vaší organizace. Pokud provádíte úkoly, doporučujeme však uvedení osobní e-mailovou adresu ve vašem účtu GitHub tak, aby vždycky budete mít přístup k němu. Jakmile to uděláte, zkuste se znovu přihlásit a zvolte **použít jinou e-mailovou adresu** Jakmile budete vyzváni k přihlášení ke svému pracovnímu nebo školnímu účtu. To vám umožní vytvořit nový účet Microsoft pomocí tohoto osobní e-mailovou adresu.
