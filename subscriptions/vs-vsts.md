- Zlepšení kvality kódu a catch problémy již v rané fázi.
- Zautomatizovat a zjednodušit nasazení Azure
- Zvýšení produktivity s výkonné funkce.

Nejste si jistí jaké předplatné používáte?  Připojení k [https://my.visualstudio.com/subscriptions](https://my.visualstudio.com/subscriptions?wt.mc_id=o~msft~docs) zobrazíte všechny odběry, které jsou přiřazeny k e-mailovou adresu. Pokud nevidíte všechny odběry, můžete mít jeden nebo více přiřadit jinou e-mailovou adresu.  Musíte se přihlásit pomocí tohoto e-mailovou adresu zobrazíte těchto předplatných. 

Při přihlášení do služby VSTS pomocí stejné identity, který jste použili pro aktivaci předplatného sady Visual Studio Uvědomujeme si to automaticky. Tento postup funguje pro primární identity, který používáte při přihlášení k portálu odběratele a žádné alternativní identitu, kterou jste nakonfigurovali pro vaše předplatné sady Visual Studio. Služby VSTS podporuje oba účty Microsoft (jako je @outlook.com) a pracovní nebo školní účty (které využívají služby Azure Active Directory spravovaná vaší organizací). Může použít k vaší primární a alternativní identit služby VSTS a mohou připojit libovolný počet účtů služby VSTS jako člena.

Pokud chcete používat služby VSTS, budete potřebovat účet. Můžete přihlásit pomocí existující účet, nebo vytvořte novou.  Chcete-li vytvořit nový účet:
1.  Přihlaste se k [https://my.visualstudio.com/benefits](https://my.visualstudio.com/benefits?wt.mc_id=o~msft~docs).
2.  Vyhledejte dlaždici Visual Studio Team Services v části nástroje a klikněte na odkaz "Začínáme" v dolní části dlaždici výhody.   

Součástí těchto funkcí služby VSTS jsou k dispozici následující odběry: 
- Visual Studio Enterprise: [základní](https://www.visualstudio.com/team-services/compare-features/), [nástroje Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web), [balíčku správy](https://marketplace.visualstudio.com/items?itemName=ms.feed)
- Visual Studio Professional: [Basic](https://www.visualstudio.com/team-services/compare-features/)
- Platformy MSDN: [základní](https://www.visualstudio.com/team-services/compare-features/), [nástroje Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)
- Visual Studio Test Professional: [základní](https://www.visualstudio.com/team-services/compare-features/), [nástroje Test Manager](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)

3.  Zadejte název pro svůj web služby VSTS projektu.  

4.  Zvolte, zda chcete použít **Git** nebo **Team Foundation verze ovládacího prvku (TFVC)** ke správě projektu.  Jde o neměnnou volbu pro každý týmový projekt, který vytvoříte, ale ve stejné kolekci týmových projektů můžete použít jak týmové projekty TFVC, tak Git.  Nejste si jisti, který se použije? 
    - Git: Git je systém správy decentralizované verzí. Každý vývojář má ve svém vývojovém počítači kopii celého zdrojového úložiště. Vývojáři mohou bez připojení k síti potvrzovat jednotlivé sady změn ve svých počítačích a provádět operace správy verzí, jako je historie a porovnávání.  [Další informace](https://www.visualstudio.com/docs/git/gitquickstart) o Git.
    - TFVC: Team Foundation verze ovládacího prvku (TFVC) je systém správy centralizované verzí. Členové týmu obvykle mají jenom jedna verze jednotlivých souborů na jejich počítačích vývojářů. Historická data se udržují pouze na serveru. Větve jsou založeny na cestě a vytvořeny na serveru. [Další informace](https://www.visualstudio.com/docs/tfvc/overview) o správy verzí Team Foundation.

 
5.  Klikněte na tlačítko **změnit podrobnosti o** přizpůsobit možnosti název projektu, jak uspořádat práci (Scrum, agilní, CMMI), kde k hostování vašich projektů a postup sdílení práce s ostatními.  Klikněte na tlačítko "Pokračovat".

# <a name="create-your-vsts-account"></a>Vytvoření účtu služby VSTS

6.  To bude trvat chvíli k vytvoření účtu a pak se zobrazí stránka služby VSTS pro svůj první projekt pomocí názvu, který jste zadali.  Nyní jste připraveni začít používat Visual Studio Team Services!

Pokud obdržíte e-mail s potvrzením, že jste úspěšně vytvořili váš účet.  Také uvádí vaše adresa URL účtu a přihlaste se a upřednostňované e-mailové adresy.
