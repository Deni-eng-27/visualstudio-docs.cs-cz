# <a name="manage-visual-studio-subscriptions-with-github-enterprise"></a>Správa předplatných Visual Studia pomocí GitHubu Enterprise
Zákazníci, kteří mají smlouvy Enterprise (EA) se společností Microsoft, si mohou zakoupit novou nabídku předplatného, která sdružuje standardní předplatná Sady Visual Studio a GitHub Enterprise. Je to snadný a ekonomický způsob, jak předplatitelé Visual Studia získat GitHub Enterprise. 

Když vaše organizace zakoupí předplatná Visual Studia s GitHub Enterprise, zřídí se a spravovají se ve dvou částech.

## <a name="manage-visual-studio-subscriptions"></a>Správa předplatných sady Visual Studio
Když vaše organizace zakoupí předplatná Sady Visual Studio s GitHub Enterprise, část předplatných sady Visual Studio se okamžitě zřídí a předplatná jsou k dispozici pro přiřazení a správu na portálu [správy předplatných](https://manage.visualstudio.com) sady Visual Studio. 

Další informace o správě předplatných najdete v těchto tématech:
- [Použití Portálu pro správu](using-admin-portal.md)
- [Přiřazení předplatných](assign-license.md)
- [Úprava předplatných](edit-license.md)
- [Odstranění předplatných](delete-license.md)
- [Nadměrná přidělení](handle-overclaimed-license.md)

> [!Important]
> Pokud předplatná Visual Studia s GitHub Enterprise jsou přiřazeny správci předplatného Visual Studia a nikdy se nenakupovaly, nebudou viditelné pro správce GitHub Enterprise v rámci organizace. Aby bylo zajištěno, že jsou viditelná předplatná GitHub Enterprise, měl by se při prvním přiřazení předplatných provést nákup včetně **alespoň jednoho** visual studia Professional s GitHub Enterprise nebo Visual Studio Enterprise s předplatným GitHub Enterprise.  
>
> Je odpovědností zákazníka zajistit, aby pro každé předplatné GitHub, které je přiřazeno, bylo odpovídající visual studio s předplatným GitHub přiřazené na portálu správy předplatných sady Visual Studio, aby zůstalo v souladu s licenční požadavky pro toto předplatné.

## <a name="manage-github-enterprise-subscriptions"></a>Správa předplatných GitHub Enterprise
Když se nakupují předplatná GitHub Enterprise, GitHub spolupracuje se zákazníky, aby pomohli vytvořit a nakonfigurovat organizace, které budou přistupovat ke GitHubu a identifikovat správce.  Tito správci pak obdrží oznámení, že byli nastaveni jako správci.  

Vzhledem k tomu, že tento proces je složitější, může trvat několik dní po zakoupení předplatného pro organizace a správce, které mají být plně nastaveny.

GitHub je k dispozici buď jako cloudová GitHub.com, nebo jako místní GitHub Enterprise Server.  Procesy pro správu dvou verzí se liší.  GitHub poskytuje řadu témat nápovědy a pokynů pro správce, které vám pomůžou spravovat předplatná GitHub Enterprise.  Níže jsme poskytli odkazy na vybraná témata.
