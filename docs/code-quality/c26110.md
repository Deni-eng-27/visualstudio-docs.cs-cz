# <a name="c26110"></a>C26110
Warning C26110: selhání volajícího pro blokování > \<lock před voláním funkce \<func >.

 Když je vyžadován zámek, nezapomeňte vyjasnit, zda by funkce nebo její volající získal zámek. Upozornění C26110 se vydá, když dojde k porušení poznámky `_Requires_lock_held_` nebo k jiným poznámkaem souvisejícím s zámkem. Další informace najdete v tématu [poznámky k chování při zamykání](annotating-locking-behavior.md) .

## <a name="example"></a>Příklad
 V následujícím příkladu je vygenerováno upozornění C26110, protože Poznámka `_Requires_lock_held_` u funkce `LockRequired` uvádí, že volající `LockRequired` musí získat zámek před voláním `LockRequired`. Bez této poznámky musí `LockRequired` získat zámek před tím, než přistupuje ke sdíleným datům chráněným zámkem.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
    int d;
} DATA;

_Requires_lock_held_(p->cs)

void LockRequired(DATA* p)
{
    p->d = 0;
}

void LockNotHeld(DATA* p)
{
    LockRequired(p); // Warning C26110
}
```
