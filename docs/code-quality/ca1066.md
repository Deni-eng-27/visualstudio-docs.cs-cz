# <a name="ca1066-implement-iequatable-when-overriding-equals"></a>CA1066: implementovat IEquatable při přepisování Equals

|||
|-|-|
|TypeName|EquatableAnalyzer|
|CheckId|CA1066|
|Kategorie|Microsoft. Design|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Hodnotový typ (struct) Přepisuje <xref:System.Object.Equals%2A> metodu, ale neimplementuje <xref:System.IEquatable%601>.

## <a name="rule-description"></a>Popis pravidla

Přepsání <xref:System.Object.Equals%2A> typu hodnoty označuje, že podporuje porovnání dvou instancí typu s hodnotou rovnosti. Zvažte implementaci <xref:System.IEquatable%601> rozhraní pro podporu testů silného typu pro rovnost. Tím se zajistí, aby volající provádělcí kontroly rovnosti volali metodu silného typu <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> a zabránili zabalení argumentu a zvýšili výkon. Další informace najdete [tady](/dotnet/api/system.iequatable-1#notes-to-implementers).

Vaše <xref:System.IEquatable%601.Equals%2A?displayProperty=fullName> implementace by měla vracet výsledky, které jsou <xref:System.Object.Equals%2A>konzistentní s.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Chcete-li opravit porušení, <xref:System.IEquatable%601> implementujte <xref:System.Object.Equals%2A> a aktualizujte přepsání pro vyvolání této implementované metody. Například následující dva fragmenty kódu ukazují porušení pravidla a jejich opravu:

```csharp
public struct S
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && _value == otherS._value;
}
```

```csharp
using System;

public struct S : IEquatable<S>
{
    private readonly int _value;
    public S(int f)
    {
        _value = f;
    }

    public override int GetHashCode()
        => _value.GetHashCode();

    public override bool Equals(object other)
        => other is S otherS && Equals(otherS);

    public bool Equals(S other)
        => _value == other._value;
}
```

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

V případě, že jsou výhody návrhu a výkonu implementace rozhraní nepostradatelné, je dobré potlačit porušení z tohoto pravidla.
