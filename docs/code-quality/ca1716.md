# <a name="ca1716-identifiers-should-not-match-keywords"></a>CA1716: Identifikátory by neměly odpovídat klíčovým slovům

|||
|-|-|
|TypeName|IdentifiersShouldNotMatchKeywords|
|CheckId|CA1716|
|Kategorie|Microsoft.Naming|
|Narušující změny|Narušující|

## <a name="cause"></a>Příčina

Název oboru názvů, typu nebo virtuálního člena nebo člena rozhraní odpovídá vyhrazenému klíčovému slovu v programovacím jazyce.

Ve výchozím nastavení se toto pravidlo dívá pouze na externě viditelné obory názvů, typy a členy, ale můžete [konfigurovat viditelnost](#api-surface) a [druhy symbolů](#analyzed-symbol-kinds).

## <a name="rule-description"></a>Popis pravidla

Identifikátory pro obory názvů, typy a členy virtuálního rozhraní a rozhraní by neměly odpovídat klíčovým slovům definovaným jazyky, které cílí na modul runtime společného jazyka. V závislosti na použitém jazyce a klíčovém slově mohou chyby kompilátoru a nejasnosti ztížit použití knihovny.

Toto pravidlo kontroluje klíčová slova v následujících jazycích:

- Visual Basic
- C#
- C++/CLI

Porovnání bez rozlišování velkých a malých písmen se používá pro klíčová slova jazyka Visual Basic a porovnání rozlišování velkých a malých písmen se používá pro ostatní jazyky.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Vyberte název, který se nezobrazuje v seznamu klíčových slov.

## <a name="when-to-suppress-warnings"></a>Kdy potlačit varování

Upozornění od tohoto pravidla můžete potlačit, pokud jste přesvědčeni, že identifikátor nezaměňuje uživatele rozhraní API a že knihovna je použitelná ve všech dostupných jazycích v rozhraní .NET.

## <a name="configurability"></a>Konfigurovatelnost

### <a name="api-surface"></a>Povrch rozhraní API

Pokud používáte toto pravidlo z [analyzátorů FxCop](install-fxcop-analyzers.md) (a ne s analýzou starší verze), můžete nakonfigurovat, na které části vašeho základu kódu chcete toto pravidlo spustit, na základě jejich usnadnění přístupu. Chcete-li například určit, že pravidlo by mělo být spuštěno pouze proti neveřejnému povrchu rozhraní API, přidejte do souboru .editorconfig v projektu následující dvojici klíč-hodnota:

```ini
dotnet_code_quality.ca1716.api_surface = private, internal
```

Tuto možnost můžete nakonfigurovat pouze pro toto pravidlo, pro všechna pravidla nebo pro všechna pravidla v této kategorii (Pojmenování). Další informace naleznete v [tématu Konfigurace analyzátorů FxCop](configure-fxcop-analyzers.md).

### <a name="analyzed-symbol-kinds"></a>Analyzované druhy symbolů

Můžete nakonfigurovat typy členů, které budou analyzovány podle tohoto pravidla. Jako seznam oddělený čárkami `Method`použijte `Property` `Event`jednu nebo více těchto hodnot `Parameter` `Namespace`, `NamedType`, , . Všimněte si, že výchozí hodnota zahrnuje všechny výše uvedené členy.

```ini
dotnet_code_quality.CA1716.analyzed_symbol_kinds = Namespace, NamedType, Method, Property, Event
```
