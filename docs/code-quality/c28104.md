# <a name="c28104"></a>C28104
Upozornění C28104: Prostředek, který by měl být získán před ukončením funkce, nebyl získán

 Funkce, která je určena k získání prostředku před tím, že byla ukončena bez získání prostředku. Toto upozornění znamená, že je funkce opatřena poznámkou `__drv_acquiresResource`, ale nevrací se s tím, že se prostředek skutečně získal. Pokud je tato funkce obálkovou funkcí, cesta prostřednictvím funkce nedosáhla zabalené funkce. Pokud se selhání dorazí na zabalenou funkci, protože funkce vrátila chybu a nezískala prostředek, možná bude nutné použít podmíněné poznámky (`__drv_when`).

 Pokud tato funkce ve skutečnosti implementuje pořízení prostředku, nemusí být možné, aby PFD zjistil, že se prostředek získal. V takovém případě použijte k potlačení chyby upozornění `#pragma`. @No__t-0 můžete umístit na řádek před `{`, který zahájí tělo funkce. Volající funkce stále potřebují anotaci, ale nástroj pro analýzu kódu nebude moci detekovat, zda byl prostředek získán.

## <a name="example"></a>Příklad

```
__drv_acquireResourceGlobal(HWLock, lockid)
void GetHardwareLock(lockid)
#pragma warning (suppress: 28104)
{
   // code to implement a hardware lock (which the Code Analysis tool can't recognize)
}
```
