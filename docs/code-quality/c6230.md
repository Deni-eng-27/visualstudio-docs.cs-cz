# <a name="c6230"></a>C6230

Upozornění C6230: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: použití HRESULT v logickém kontextu

Toto upozornění indikuje, že `HRESULT` se používá v kontextu, kde je očekáván logický výsledek, jako je například příkaz `if`. Tento test bude nejspíš vracet nesprávné výsledky. Například hodnota typického úspěchu pro `HRESULT` (`S_OK`) má hodnotu false, pokud je testována jako logická hodnota.

## <a name="example"></a>Příklad

Následující kód vygeneruje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (hr)
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Ve většině případů by mělo být k otestování hodnoty `HRESULT` použito makro [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) nebo [Failed](/windows/desktop/api/winerror/nf-winerror-failed) . Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
#include <windows.h>

VOID f( )
{
  LPMALLOC pMalloc;
  HRESULT hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {

    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pro toto upozornění je typ `SCODE` považován za `HRESULT`.

Použití `malloc` a `free` (a souvisejících rozhraní API dynamické paměti) má mnoho nástrah jako příčinu nevracení paměti a výjimek. Chcete-li se vyhnout těmto druhům úniku a potížím s výjimkami, použijte ukazatele a C++ třídy kontejneru poskytované standardní knihovnou. Patří mezi ně [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)a [Vector](/cpp/standard-library/vector). Další informace najdete v tématu [inteligentní ukazatele](/cpp/cpp/smart-pointers-modern-cpp) a [ C++ standardní knihovna](/cpp/standard-library/cpp-standard-library-reference).
