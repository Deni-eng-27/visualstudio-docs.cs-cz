# <a name="ca3061-do-not-add-schema-by-url"></a>CA3061: Nepřidávat schéma podle adresy URL

|||
|-|-|
|CheckId|CA3061|
|Kategorie|Microsoft.Security|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Přetížení používá `XmlSchemaCollection.Add(String, String)` `XmlUrlResolver` k určení externího schématu XML ve formě identifikátoru URI. Pokud je řetězec identifikátoru URI v kódu, může to vést k analýze škodlivého schématu XML, které umožňuje zahrnutí bomby XML a škodlivých externích entit. To může útočníkovi umožnit způsobit odepření služby, zpřístupnění informací nebo útok na vyžádání na straně serveru.

## <a name="rule-description"></a>Popis pravidla

Nepoužívejte nezabezpečené přetížení `Add` metody, protože by mohlo dojít k nebezpečným externím odkazům.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

- Nepoužívejte `XmlSchemaCollection.Add(String, String)` .

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Toto pravidlo potlačit, pokud si nejste jisti, že váš kód XML neřeší nebezpečné externí odkazy.

## <a name="pseudo-code-examples"></a>Příklady kódu pseudo

### <a name="violation"></a>Selhání

Následující ukázka pseudo kódu znázorňuje vzor zjištěný tímto pravidlem.
Druhý typ parametru je `string` .

```csharp
using System;
using System.Xml.Schema;
...
XmlSchemaCollection xsc = new XmlSchemaCollection();
xsc.Add("urn: bookstore - schema", "books.xsd");
```

### <a name="solution"></a>Řešení

```csharp
using System;
using System.IO;
using System.Xml;
using System.Xml.Schema;
...
XmlSchemaCollection xsc = new XmlSchemaCollection();
xsc.Add("urn: bookstore - schema", new XmlTextReader(new FileStream(""xmlFilename"", FileMode.Open)));
```
