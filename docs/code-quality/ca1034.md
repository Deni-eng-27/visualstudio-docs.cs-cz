# <a name="ca1034-nested-types-should-not-be-visible"></a>CA1034: Vnořené typy by neměly být viditelné

|||
|-|-|
|TypeName|NestedTypesShouldNotBeVisible|
|CheckId|CA1034|
|Kategorie|Microsoft.Design|
|Narušující změny|Narušující|

## <a name="cause"></a>Příčina

Externě viditelný typ obsahuje externě viditelné deklarace typu. Vnořené výčty, chráněné typy a tvůrce vzory jsou osvobozeny od tohoto pravidla.

## <a name="rule-description"></a>Popis pravidla
Vnořený typ je typ deklarovaný v rámci oboru jiného typu. Vnořené typy jsou užitečné pro zapouzdření podrobnosti soukromé implementace obsahující typ. Jsou-li vnořené typy používány za tímto účelem, neměly by být externě viditelné.

Nepoužívejte externě viditelné vnořené typy pro logické seskupení nebo aby se zabránilo kolizím názvů; místo toho použijte obory názvů.

Vnořené typy zahrnují pojem usnadnění přístupu členů, které někteří programátoři nerozumí jasně.

Chráněné typy lze použít v podtřídách a vnořených typech ve scénářích předběžného přizpůsobení.

## <a name="how-to-fix-violations"></a>Jak opravit porušení
Pokud nechcete, aby byl vnořený typ externě viditelný, změňte přístupnost typu. V opačném případě odeberte vnořený typ z nadřazeného typu. Pokud je účelem vnoření kategorizace vnořeného typu, použijte místo toho obor názvů k vytvoření hierarchie.

## <a name="when-to-suppress-warnings"></a>Kdy potlačit varování
Nepotlačujte upozornění na toto pravidlo.
