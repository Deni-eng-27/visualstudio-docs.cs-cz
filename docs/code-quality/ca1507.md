# <a name="ca1507-use-nameof-in-place-of-string"></a>CA1507: místo řetězce použijte `nameof`.

|||
|-|-|
|TypeName|UseNameofInPlaceOfString|
|CheckId|CA1507|
|Kategorie|Microsoft. udržovatelnost|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>příčina

Jako argument metody je použit literál `string` nebo konstanta, která se shoduje s názvem parametru obsahující metody nebo názvem vlastnosti obsahujícího typu.

## <a name="rule-description"></a>Popis pravidla

Pravidlo CA1507 označuje použití literálu `string` jako argument metody nebo konstruktoru, kde výraz [nameof](/dotnet/csharp/language-reference/keywords/nameof) (`NameOf` in Visual Basic) by mohl přidat udržovatelnost. Pravidlo se aktivuje, pokud jsou splněné všechny následující podmínky:

- Argument je `string` literál nebo konstanta.

- Hodnota konstanty `string` literálu se shoduje s jedním z následujících způsobů:

  - Název parametru metody, výrazu lambda nebo místní funkce, ve které je vyvolána metoda nebo konstruktor.

  - Název vlastnosti typu, ve kterém je vyvolána metoda nebo konstruktor.

- Argument odpovídá parametru `string`ho typu metody nebo konstruktoru, který je vyvolán (to znamená, že na webu volání není žádný převod).

CA1507 pravidel vylepšuje udržovatelnost kódu v případech, kdy parametr může být v budoucnu přejmenován, ale literál `string` je omylem přejmenován. Když použijete `nameof`, symbol se přejmenuje, když se parametr přejmenuje pomocí operace refaktoringu. Kromě toho všechny pravopisné chyby v názvu parametru jsou zachyceny kompilátorem.

> [!NOTE]
> Pravidlo CA1507 není k dispozici v analýze starší verze. Poprvé jsme zavedli v [analyzátorech FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) verze 2.6.0.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Chcete-li opravit porušení, nahraďte `string` literál výrazem [nameof](/dotnet/csharp/language-reference/keywords/nameof) (`NameOf` in Visual Basic). Například následující dva fragmenty kódu ukazují porušení pravidla a jejich opravu:

```csharp
public Book(string title)
{
    // Violates rule CA1507
    Title = title ?? throw new ArgumentNullException("title", "All books must have a title.");
}
```

```csharp
public Book(string title)
{
    // Resolves rule CA1507 violation
    Title = title ?? throw new ArgumentNullException(nameof(title), "All books must have a title.");
}
```

> [!TIP]
> Pro toto pravidlo v aplikaci Visual Studio je k dispozici oprava kódu. Pokud ho chcete použít, umístěte kurzor na `string` literál a stiskněte **Ctrl** + **.** (tečka). V seznamu zobrazených možností vyberte **použít nameof k vyjádření názvů symbolů** .
>
> ![Oprava kódu pro CA1507 – použití nameof k vyjádření názvů symbolů](media/ca1507-code-fix.PNG)

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Pokud nejste v starosti s údržbou vašeho kódu, je bezpečné potlačit porušení tohoto pravidla.
