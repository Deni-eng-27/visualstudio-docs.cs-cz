# <a name="c26401-dont_delete_non_owner"></a>C26401 DONT_DELETE_NON_OWNER
Tato kontrolu detekuje místa, kde přechod na `owner<T>` může být vhodnou možností pro první fázi refaktoringu. Podobně jako C26400 vynutila pravidla I. 11 a R. 3, ale zaměřuje se na část Release životnosti ukazatele. Upozorňuje na jakékoli volání operátoru `delete`, pokud jeho cíl není `owner<T>` ani implicitně se nepředpokládá vlastník. Další informace najdete v tématu [C26400](c26400.md) týkajícím se automatických deklarací. To zahrnuje výrazy, které odkazují na globální proměnné, formální a tak dále.

Upozornění C26400 a C26401 se vždy vyskytují u [C26409](c26409.md), ale jsou vhodnější pro scénáře, kdy není možné okamžitě migrovat na inteligentní ukazatele. V takových případech je možné nejprve přijmout koncept `owner<T>` a C26409 může být dočasně potlačený.
