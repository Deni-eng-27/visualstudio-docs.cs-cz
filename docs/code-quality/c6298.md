# <a name="c6298"></a>C6298
upozornění C6298: řetězec jen pro čtení pomocí \<ukazatel > jako argument řetězec zapisovatelné: to se pokusí o zápis do statické paměti a způsobit náhodné havárií

 Toto upozornění se označuje použití konstantní řetězec jako argument pro funkci, která může upravovat obsah tento řetězec. Vzhledem k tomu, kompilátor konstantní řetězců v statické paměti, všechny pokusy o upravit způsobit narušení přístupu a náhodných dojde k chybě.

 To může zabránit ukládání konstantní řetězec do místní pole a pak pomocí pole jako argument funkce.

## <a name="example"></a>Příklad
 Následující vzorový kód generuje toto upozornění:

```
#include <windows.h>
#include <stdio.h>

void f()
{
    STARTUPINFO si;
    PROCESS_INFORMATION pi;

    ZeroMemory( &si, sizeof( si ) );
    si.cb = sizeof( si );
    ZeroMemory( &pi, sizeof( pi ) );
    if( !CreateProcess(NULL,
                      "\"c:\\Windows\\system32\\calc.exe\"",
                      NULL,
                      NULL,
                      FALSE,
                      0,
                      NULL,
                      NULL,
                      &si,
                      &pi ) )
    {
        puts( "CreateProcess failed." );
        return;
    }

    // Wait until child process exits.
    WaitForSingleObject( pi.hProcess, INFINITE );

    // Close process and thread handles.
    CloseHandle( pi.hProcess );
    CloseHandle( pi.hThread );
}

```

 Chcete-li toto upozornění, použijte následující vzorový kód:

```
#include <windows.h>
#include <stdio.h>

void f( )
{
    STARTUPINFO si;
    PROCESS_INFORMATION pi;

    char szCmdLine[] = "\"c:\\Windows\\system32\\calc.exe\"";
    ZeroMemory( &si, sizeof( si ) );
    si.cb = sizeof( si );
    ZeroMemory( &pi, sizeof( pi ) );

    if( !CreateProcess(NULL,
                      szCmdLine,
                      NULL,
                      NULL,
                      FALSE,
                      0,
                      NULL,
                      NULL,
                      &si,
                      &pi ) )
    {
        puts( "CreateProcess failed." );
        return;
    }

    // Wait until child process exits.
    WaitForSingleObject( pi.hProcess, INFINITE );

    // Close process and thread handles.
    CloseHandle( pi.hProcess );
    CloseHandle( pi.hThread );
}
```
