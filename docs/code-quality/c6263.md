# <a name="c6263"></a>C6263
Upozornění C6263: použití _alloca ve smyčce; To může rychle přetečení zásobníku

 Toto upozornění signalizuje, že volání _alloca uvnitř smyčky k přidělení paměti může způsobit přetečení zásobníku. _alloca přiděluje paměť ze zásobníku, ale paměť je uvolněna pouze v případě, že volání funkce skončí. Zásobník je i v uživatelském režimu omezený a selhání při potvrzení stránky zásobníku způsobí výjimku přetečení zásobníku. Funkce `_resetstkoflw` se obnoví z podmínky přetečení zásobníku, což umožní programu pokračovat, aniž by došlo k selhání s závažnou chybou výjimky. Pokud není volána funkce `_resetstkoflw`, není po předchozí výjimce žádná stránka Guard. Při příštím přetečení zásobníku nejsou vůbec žádné výjimky a proces se ukončí bez upozornění.

 Vyhněte se volání `_alloca` uvnitř smyčky, pokud je velikost alokace nebo počet iterací neznámý, protože by mohlo dojít k přetečení zásobníku. V těchto případech zvažte další možnosti, jako například, paměť haldy nebo [ C++ třídy standardní knihovny](/cpp/standard-library/cpp-standard-library-reference) .

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění:

```
#include <windows.h>
#include <malloc.h>
#include <excpt.h>
#include <stdio.h>

#define MAX_SIZE 50

void f ( int size )
{
  char* cArray;
  __try
  {
    for(int i = 0; i < MAX_SIZE; i++)
    {
      cArray = (char *)_alloca(size);

     // process cArray...
    }
  }
  __except(GetExceptionCode() == STATUS_STACK_OVERFLOW ?
              EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
     // code...
     puts("Allocation Failed");
    _resetstkoflw();
  }
}
```

 Následující kód používá k opravě tohoto upozornění stav \ ():

```
#include <windows.h>
#define MAX_SIZE 50

void f ( int size )
{
  char* cArray;

  for(int i = 0; i < MAX_SIZE; i++)
  {
     cArray = (char *) malloc(size);
     if (cArray != NULL)
     {
       // process cArray...
       free(cArray);
     }
  }
}
```
