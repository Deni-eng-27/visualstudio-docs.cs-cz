# <a name="ca1827-do-not-use-countlongcount-when-any-can-be-used"></a>CA1827: Nepoužívejte Count/LongCount, když se dá použít Any.

|||
|-|-|
|TypeName|DoNotUseCountWhenAnyCanBeUsedAnalyzer|
|CheckId|CA1827|
|Kategorie|Microsoft. Performance|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Metoda <xref:System.Linq.Enumerable.Count%2A> nebo <xref:System.Linq.Enumerable.LongCount%2A> byla použita, kde <xref:System.Linq.Enumerable.Any%2A> by byla metoda efektivnější.

## <a name="rule-description"></a>Popis pravidla

Toto pravidlo označuje volání <xref:System.Linq.Enumerable.Count%2A> metody <xref:System.Linq.Enumerable.LongCount%2A> a, která slouží ke kontrole, zda má kolekce alespoň jeden prvek. Tato volání metod vyžadují vytvoření výčtu celé kolekce pro výpočet počtu. Stejná kontrolu je rychlejší s <xref:System.Linq.Enumerable.Any%2A> metodou, protože se vyhne vytváření výčtu kolekce.

> [!NOTE]
> Pravidlo CA1827 není k dispozici v analýze starší verze. Poprvé jsme zavedli v [analyzátorech FxCop](https://www.nuget.org/packages/Microsoft.CodeAnalysis.FxCopAnalyzers) verze 2.9.6.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Chcete-li opravit porušení, <xref:System.Linq.Enumerable.Count%2A> nahraďte <xref:System.Linq.Enumerable.LongCount%2A> volání <xref:System.Linq.Enumerable.Any%2A> metody nebo metodou. Například následující dva fragmenty kódu ukazují porušení pravidla a jejich opravu:

```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Count() != 0 ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.LongCount() > 0 ? "Not empty" : "Empty";
}
```


```csharp
using System.Collections.Generic;
using System.Linq;

class C
{
    public string M1(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";

    public string M2(IEnumerable<string> list)
        => list.Any() ? "Not empty" : "Empty";
}
```

> [!TIP]
> Pro toto pravidlo v aplikaci Visual Studio je k dispozici oprava kódu. Pokud ho chcete použít, umístěte kurzor na porušení pole a stiskněte **klávesu CTRL**+**.** (tečka). Možnost **Nepoužívat Count () ani LongCount (), pokud lze použít Any ()** , ze seznamu zobrazených možností.
>
