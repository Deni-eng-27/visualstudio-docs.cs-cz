# <a name="c6318"></a>C6318
Upozornění C6318: Nesprávně definovaný __try/@no__t -0 _except: použití konstanty EXCEPTION_CONTINUE_SEARCH nebo jiné konstanty, která se ve výrazu filtru výjimky vyhodnocuje jako nula. Kód v bloku obslužné rutiny výjimky není proveden.

 Toto upozornění znamená, že pokud dojde k výjimce v chráněném bloku této strukturované obslužné rutiny výjimek, výjimka nebude zpracována, protože ve výrazu filtru výjimky je použit konstanta `EXCECPTION_CONTINUE_SEARCH`.

 Tento kód je ekvivalentní chráněnému bloku bez bloku obslužných rutin výjimek, protože blok obslužné rutiny není spuštěn.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění:

```
#include <excpt.h>
#include <stdio.h>

void f (char *pch)
{
  __try
  {
    // assignment might fail
    *pch = 0;
  }
  __except (EXCEPTION_CONTINUE_SEARCH) // warning 6318
  {
    puts("Exception Occurred");
  }
}
```

 Chcete-li toto upozornění opravit, použijte následující kód:

```
#include <excpt.h>
#include <stdio.h>
#include <windows.h>

void f (char *pch)
{
  __try
  {
    // assignment might fail
    *pch = 0;
  }
  __except( (GetExceptionCode() == EXCEPTION_ACCESS_VIOLATION) ?
                EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH )
  {
    puts("Access violation");
  }
}
```
