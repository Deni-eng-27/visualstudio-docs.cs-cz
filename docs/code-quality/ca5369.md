# <a name="ca5369-use-xmlreader-for-deserialize"></a>CA5369: použít XmlReader pro deserializaci

|||
|-|-|
|TypeName|UseXMLReaderForDeserialize|
|CheckId|CA5369|
|Kategorie|Microsoft.Security|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>příčina

Deserializace nedůvěryhodného vstupu XML s <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> vytvořena instance bez objektu `XmlReader` může potenciálně vést k odepření útoků služby, zpřístupnění informací a útokům na serverové požadavky na straně serveru. Tyto útoky jsou povoleny nedůvěryhodnou verzí DTD a zpracování schématu XML, což umožňuje zahrnutí bomby XML a škodlivých externích entit do XML. Pouze pomocí `XmlReader` je možné zakázat DTD. Vložené zpracování schématu XML jako `XmlReader` má vlastnost `ProhibitDtd` a `ProcessInlineSchema` nastavenou na `false` ve výchozím nastavení ve .NET Framework verze 4,0 a novější. Další možnosti, jako `Stream`, `TextReader`a `XmlSerializationReader`, nemůžou zakázat zpracování DTD.

## <a name="rule-description"></a>Popis pravidla

Zpracování nedůvěryhodných schémat DTD a XML může povolit načítání nebezpečných externích odkazů, které by měly být omezeny pomocí `XmlReader` se zabezpečeným překladačem nebo se zakázaným zpracováním vloženého schématu DTD a XML. Toto pravidlo detekuje kód, který používá metodu <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> a nepřijímá `XmlReader` jako parametr konstruktoru.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Nepoužívejte <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> přetížení kromě <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader)>, <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String)>, <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.Xml.Serialization.XmlDeserializationEvents)>nebo <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String,System.Xml.Serialization.XmlDeserializationEvents)>.

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Toto upozornění můžete potenciálně potlačit, pokud analyzovaný kód XML pochází z důvěryhodného zdroje, a proto nemůže být úmyslně poškozen.

## <a name="pseudo-code-examples"></a>Příklady kódu pseudo

### <a name="violation"></a>Selhání

Následující ukázka pseudo kódu znázorňuje vzor zjištěný tímto pravidlem.
Typ prvního parametru <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> není `XmlReader` nebo odvozenou třídou.

```csharp
using System.IO;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass).Deserialize(new FileStream("filename", FileMode.Open));
```

### <a name="solution"></a>Řešení

```csharp
using System.IO;
using System.Xml;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass)).Deserialize(XmlReader.Create (new FileStream("filename", FileMode.Open)));
```
