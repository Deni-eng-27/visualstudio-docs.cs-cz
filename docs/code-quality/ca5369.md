# <a name="ca5369-use-xmlreader-for-deserialize"></a>CA5369: Použít XmlReader pro Deserialize

|||
|-|-|
|CheckId|CA5369|
|Kategorie|Microsoft.Security|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Deserializace nedůvěryhodného vstupu XML s <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> vytvořením instance bez `XmlReader` objektu může potenciálně vést k odepření služby, zpřístupnění informací a útokům na straně serveru pro falšování požadavků. Tyto útoky jsou povoleny nedůvěryhodnou verzí DTD a zpracování schématu XML, což umožňuje zahrnutí bomby XML a škodlivých externích entit do XML. Pouze pomocí `XmlReader` je možné zakázat DTD. Vložené schéma XML je zpracováváno stejně jako `XmlReader` `ProhibitDtd` vlastnost a ve `ProcessInlineSchema` `false` výchozím nastavení nastaveno na .NET Framework verze 4,0 a novější. Další možnosti, jako například `Stream` , `TextReader` , a `XmlSerializationReader` nelze zakázat zpracování DTD.

## <a name="rule-description"></a>Popis pravidla

Zpracování nedůvěryhodných schémat DTD a XML může povolit načítání nebezpečných externích odkazů, které by měly být omezeny pomocí `XmlReader` zabezpečeného překladače nebo se zakázaným zpracováním vloženého schématu DTD a XML. Toto pravidlo detekuje kód, který používá <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> metodu a nepřijímá se `XmlReader` jako parametr konstruktoru.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Nepoužívejte <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> přetížení, které není,, <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader)> <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String)> <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.Xml.Serialization.XmlDeserializationEvents)> nebo <xref:System.Xml.Serialization.XmlSerializer.Deserialize(System.Xml.XmlReader,System.String,System.Xml.Serialization.XmlDeserializationEvents)> .

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Toto upozornění můžete potenciálně potlačit, pokud analyzovaný kód XML pochází z důvěryhodného zdroje, a proto nemůže být úmyslně poškozen.

## <a name="pseudo-code-examples"></a>Příklady kódu pseudo

### <a name="violation"></a>Selhání

Následující ukázka pseudo kódu znázorňuje vzor zjištěný tímto pravidlem.
Typ prvního parametru <xref:System.Xml.Serialization.XmlSerializer.Deserialize%2A?displayProperty=nameWithType> není `XmlReader` ani odvozená třída.

```csharp
using System.IO;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass).Deserialize(new FileStream("filename", FileMode.Open));
```

### <a name="solution"></a>Řešení

```csharp
using System.IO;
using System.Xml;
using System.Xml.Serialization;
...
new XmlSerializer(typeof(TestClass)).Deserialize(XmlReader.Create (new FileStream("filename", FileMode.Open)));
```
