# <a name="c26408-no_malloc_free"></a>C26408 NO_MALLOC_FREE
Toto upozornění označuje místo, kde `malloc` nebo `free` jsou vyvolány explicitně v souladu s R. 10: Vyhněte se `malloc` a `free`. Jednou z možných oprav těchto upozornění je použití [std:: make_unique](/cpp/standard-library/memory-functions#make_unique) , aby se zabránilo explicitnímu vytváření a zničení objektů. Pokud taková oprava není přijatelná, měli byste upřednostnit operátor [New a DELETE](/cpp/cpp/new-and-delete-operators) . V některých případech platí, že pokud nejsou výjimky úvodní, `malloc` a `free` lze nahradit verzí operátorů, které jsou operátory `new` a `delete`.

## <a name="remarks"></a>Poznámky
- K detekci metody \ () zkontrolujeme, jestli volání vyvolá globální funkci s názvem "název", nebo "std:: pro\". Funkce musí vrátit ukazatel na `void` a přijmout jeden parametr integrálního typu bez znaménka.
- K detekci bezplatného () kontrolujeme globální funkce s názvy "Free" nebo "std:: Free", které nevrací žádný výsledek, a přijímají jeden parametr, což je ukazatel na `void`.
