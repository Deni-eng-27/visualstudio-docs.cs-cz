# <a name="c26115"></a>C26115
Upozornění C26115: Nepodařilo se uvolnit zámek \<lock > ve funkci \<func >.

 V jazyce není prováděno vynucování dvojic s syntakticky vymezenými zámkyC++ *a jejich zámky* pro *verze* v C/programu. Funkce může zavést vedlejší efekt uzamčení tím, že provede pozorovatelnou úpravu stavu souběžnosti. Například funkce uzamknout obálka zvyšuje počet pořizování zámků nebo počet zámků pro daný zámek.

 Pomocí `_Acquires_lock_` nebo `_Releases_lock_` můžete opatřit poznámkami funkci, která má vedlejší efekt z zámku získat nebo uzamknout. Bez takovýchto poznámek se očekává, že funkce nemění počet zámků poté, co se vrátí. Pokud získání a vydání nejsou vyvážené, považují se za *osamocené*. Upozornění C26115 se vydá, když funkce zavádí osamocený zámek.

## <a name="example"></a>Příklad
 Následující příklad generuje upozornění C26115, protože existuje osamocený zámek ve funkci, která není opatřena poznámkou `_Acquires_lock_`.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

void FailToReleaseLock(int flag, DATA* p)
{
    EnterCriticalSection(&p->cs);

    if (flag)
        return; // Warning C26115

    LeaveCriticalSection(&p->cs);
}
```
