# <a name="c6221"></a>C6221
Upozornění C6221: Implicitní přetypování mezi sémanticky odlišnými typy celých čísel: porovnává HRESULT s celým číslem. Zvažte místo toho použití makra SUCCEEDED nebo FAILed.

 Toto upozornění indikuje, že `HRESULT` se porovnává s celým číslem jiným než nula. Úspěch v `HRESULT` (`S_OK`) je reprezentován hodnotou 0. Implicitní přetypování `HRESULT` na celé číslo proto vygeneruje nesprávnou hodnotu a může vést k nesprávnému výsledku. Tato funkce je často způsobena omylem, který vrací celé číslo, když ve skutečnosti vrátí `HRESULT`.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění porovnáním `HRESULT` s celočíselnou hodnotou:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == 4)
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Chcete-li toto upozornění opravit, následující kód používá makro `FAILED`:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return S_FALSE;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Pro toto upozornění je typ `SCODE` ekvivalentem `HRESULT`.
