# <a name="c6225"></a>C6225

Warning C6225: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: přiřazení 1 nebo TRUE k HRESULT. Zvažte místo toho použití S_FALSE

Toto upozornění indikuje, že `HRESULT` se přiřazuje nebo inicializuje s hodnotou explicitního 1. Logické typy označují úspěch nenulovou hodnotou; úspěch (`S_OK`) v `HRESULT` je označen hodnotou 0. Toto upozornění je často způsobeno náhodnou záměnou logických hodnot a typů `HRESULT`. Pro indikaci úspěchu by měla být použita symbolické konstanta `S_OK`.

## <a name="example"></a>Příklad

V následujícím kódu přiřazení `HRESULT` vygeneruje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

Pro toto upozornění je typ `SCODE` ekvivalentem `HRESULT`.

Pro indikaci selhání by se měla použít `E_FAIL` nebo jiná konstanta.
