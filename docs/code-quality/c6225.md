# <a name="c6225"></a>C6225

upozornění C6225: Implicitní přetypování mezi sémanticky rozdílnými celočíselnými typy: přiřazování 1 nebo TRUE proměnné HRESULT. Zvažte raději použití S_FALSE

Toto upozornění označuje, že `HRESULT` byl přiřazen nebo inicializován s explicitní hodnotou 1. Logické typy označují úspěch nenulovou hodnotou; úspěch (`S_OK`) v `HRESULT` je označen hodnotou 0. Toto upozornění je často způsobeno náhodnou záměnou typů Boolean a `HRESULT`. K označení úspěšného provedení by měla být použita symbolická konstanta `S_OK`.

## <a name="example"></a>Příklad

V následujícím kódu přiřazení `HRESULT` generuje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
  }
  else
  {
    hr = 1;
  }
}
```

Pro opravu tohoto upozornění použijte následující kód:

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
  }
  else
  {
    hr = S_FALSE;
  }
}
```

Pro toto upozornění je typ `SCODE` ekvivalentní k `HRESULT`.

K označení selhání by mělo být použito `E_FAIL` nebo jiná konstanta.
