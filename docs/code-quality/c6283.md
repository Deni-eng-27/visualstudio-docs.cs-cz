# <a name="c6283"></a>C6283
Warning C6283: > \<variable se přidělí pomocí Array New [], ale odstraní se pomocí skalárního odstranění.

 Toto upozornění se zobrazí pouze C++ v kódu a označuje, že volání funkce má nekonzistentně přidělenou paměť s operátorem `new []`, ale uvolněnou skalárním operátorem `delete`. Tato vada může způsobit nevracení, poškození paměti a v situacích, kdy byly operátory přepsány, dojde k chybě. Pokud je paměť přidělená pomocí pole `new []`, měla by být obvykle uvolněná pomocí pole `delete[]`.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete str;
}
```

 Chcete-li toto upozornění opravit, použijte následující kód:

```
void f( )
{
  char *str = new char[50];
  // code ...
  delete[] str;
}
```

 Upozornění C6283 platí pouze pro pole primitivních typů, například celá čísla nebo znaky. Pokud jsou prvky pole objekty typu třídy, je vydána výstraha [C6278](../code-quality/c6278.md) .

 Použití New a DELETE má mnoho nástrah z pohledu nevracení paměti a výjimek. Chcete-li se vyhnout těmto typům nevrácených chyb a potížím s výjimkami, použijte C++ mechanismy, které jsou k dispozici pomocí knihovny Standard Template Library (STL). Patří mezi ně [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)a [Vector](/cpp/standard-library/vector). Další informace najdete v tématu [inteligentní ukazatele](/cpp/cpp/smart-pointers-modern-cpp) a [ C++ standardní knihovna](/cpp/standard-library/cpp-standard-library-reference).
