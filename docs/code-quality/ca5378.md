# <a name="ca5378-do-not-disable-servicepointmanagersecurityprotocols"></a>CA5378: Nezakazujte ServicePointManagerSecurityProtocols

|||
|-|-|
|TypeName|DoNotSetSwitch|
|CheckId|CA5378|
|Category|Microsoft.Security|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Volání metody nastaví `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` na `true`. <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType>

## <a name="rule-description"></a>Popis pravidla

Nastavení `Switch.System.ServiceModel.DisableUsingServicePointManagerSecurityProtocols` pro`true` omezení připojení TLS (Transport Layer Security) technologie Windows Communication Framework (TLS) k použití protokolu TLS 1,0. Tato verze TLS bude zastaralá. Další informace najdete v tématu [osvědčené postupy TLS (Transport Layer Security) s .NET Framework](/dotnet/framework/network-programming/tls#switchsystemservicemodeldisableusingservicepointmanagersecurityprotocols).

## <a name="how-to-fix-violations"></a>Jak opravit porušení

- Pokud vaše aplikace cílí .NET Framework v 4.7 nebo novějším, můžete buď odebrat <xref:System.AppContext.SetSwitch%2A?displayProperty=nameWithType> volání metody, nebo nastavit hodnotu přepínače na. `false`
- Pokud je vaše aplikace cílena .NET Framework v 4.6.2 nebo starším a běží na .NET Framework v 4.7 nebo novějším, nastavte hodnotu `false`přepínače na.
- V opačném případě pro zmírnění rizik použijte [osvědčené postupy protokolu TLS (Transport Layer Security) .NET Framework](/dotnet/framework/network-programming/tls) .

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Toto upozornění můžete potlačit, pokud se potřebujete připojit ke službě starší verze, kterou nelze upgradovat, aby používala zabezpečené konfigurace TLS.

```csharp

    }
```
