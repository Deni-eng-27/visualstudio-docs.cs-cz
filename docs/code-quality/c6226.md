# <a name="c6226"></a>C6226

Upozornění C6226: Implicitní přetypování mezi sémanticky odlišnými typy celých čísel: přiřazení-1 k HRESULT. Místo toho zvažte použití E_FAIL.

Toto upozornění signalizuje, že `HRESULT` je přiřazen nebo inicializován na explicitní hodnotu-1. Toto upozornění je často způsobeno náhodnou záměněm typu Integer a `HRESULT`. Pro indikaci úspěchu použijte místo toho symbolovou konstantu `S_OK`. Pro indikaci selhání použijte symbolické konstanty, které začínají*konstantou*E_, například `E_FAIL`.

Další informace najdete v tématu [úspěšná](/windows/desktop/api/winerror/nf-winerror-succeeded) a [neúspěšná](/windows/desktop/api/winerror/nf-winerror-failed) makra.

## <a name="example"></a>Příklad

Následující kód vygeneruje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pro toto upozornění je typ `SCODE` považován za `HRESULT`.

Použití `malloc` a `free` (a souvisejících rozhraní API dynamické paměti) má mnoho nástrah jako příčinu nevracení paměti a výjimek. Chcete-li se vyhnout těmto druhům úniku a potížím s výjimkami, použijte ukazatele a C++ třídy kontejneru poskytované standardní knihovnou. Patří mezi ně [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)a [Vector](/cpp/standard-library/vector). Další informace najdete v tématu [inteligentní ukazatele](/cpp/cpp/smart-pointers-modern-cpp) a [ C++ standardní knihovna](/cpp/standard-library/cpp-standard-library-reference).
