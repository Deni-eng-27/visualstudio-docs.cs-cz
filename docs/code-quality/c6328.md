# <a name="c6328"></a>C6328

Upozornění C6328: Neshoda velikosti \<: typ > předaný \<jako číslo parametru \<>, pokud se ve volání \<funkce vyžaduje typ >. >

Předání argumentu typu `char` do modulu runtime jazyka C, rutiny založené na znacích `is<xxx>()`s názvem, `isspace()`například, mohou mít nepředvídatelné výsledky. Například znak jednobajtové hodnoty SBCS nebo MBCS typu `char` s hodnotou větší než `0x7F` je záporná hodnota. Pokud je předán, kompilátor může převést hodnotu na signed `int` nebo signed `long`. `char` Tuto hodnotu může kompilátor rozšířit a s neočekávanými výsledky. Například `isspace` akceptuje argument typu `int`, ale platný rozsah hodnot pro jeho vstupní argument je:

`0 <= c <= 255`a navíc se jedná o `EOF`speciální hodnotu.

## <a name="example"></a>Příklad

Ve výchozím nastavení `char` je ve vizuálním C++typu podepsaný typ, takže rozsah hodnot proměnné typu char je `-128 <= c <= 127`. Proto, pokud jste provedli následující `c` , by se znaménko přihlásilo `int` na znaménko s hodnotou-37, která je mimo platný rozsah pro [Space](/cpp/standard-library/locale-functions#isspace):

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

Chcete-li tento problém vyřešit, můžete `static_cast`použít, jak je znázorněno v následujícím kódu:

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

K zachycení této chyby existuje upozornění C6328. U znaků v 7 bitů rozsahu ASCII je přetypování zbytečné. Znaky mimo tento rozsah mohou mít nepředvídatelné výsledky, například Chyba programu a ukončení.
