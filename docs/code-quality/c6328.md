# <a name="c6328"></a>C6328

Upozornění C6328: Neshoda velikosti: \<type > předat jako parametr \<number >, když se \<type > vyžaduje při volání \<function >

Předání argumentu typu `char` modulu C Runtime, rutiny založené na znacích s názvem `is<xxx>()`, například `isspace()`, mohou mít nepředvídatelné výsledky. Například jednobajtové znak SBCS nebo MBCS typu `char` s hodnotou větší než `0x7F` je záporná hodnota. Pokud je předán `char`, kompilátor může převést hodnotu na podepsaný `int` nebo podepsaný `long`. Tuto hodnotu může kompilátor rozšířit a s neočekávanými výsledky. Například `isspace` přijímá argument typu `int`; platný rozsah hodnot pro vstupní argument je však následující:

`0 <= c <= 255` a navíc speciální hodnotu `EOF`.

## <a name="example"></a>Příklad

Ve výchozím nastavení je `char` podepsaný typ ve vizuálu C++, takže rozsah hodnot proměnné typu char je `-128 <= c <= 127`. Proto pokud jste provedli následující, `c` by se přihlásilo k podepsané `int` s hodnotou-37, která je mimo platný rozsah pro [místo](/cpp/standard-library/locale-functions#isspace):

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( c );
    // code ...
}
```

Chcete-li tento problém vyřešit, můžete použít `static_cast`, jak je znázorněno v následujícím kódu:

```cpp
#include <iostream>

void f( )
{
    char c = -37;
    int retVal = isspace( static_cast<unsigned char> (c) );
    // code ...
}
```

K zachycení této chyby existuje upozornění C6328. U znaků v 7 bitů rozsahu ASCII je přetypování zbytečné. Znaky mimo tento rozsah mohou mít nepředvídatelné výsledky, například Chyba programu a ukončení.
