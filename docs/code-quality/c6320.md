# <a name="c6320"></a>C6320
Upozornění C6320: výraz filtru výjimky je konstanta EXCEPTION_EXECUTE_HANDLER. To může maskovat výjimky, které nebyly určeny ke zpracování.

 Toto upozornění označuje vedlejší účinek použití konstanty EXCEPTION_EXECUTE_HANDLER v bloku __except. V tomto případě se příkaz v bloku \__except vždy spustí pro zpracování výjimky, včetně výjimek, které nechcete zpracovat v konkrétní funkci. Doporučujeme, abyste před manipulací výjimku kontrolovali.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění, protože blok __except se pokusí zpracovat výjimky všech typů:

```
#include <stdio.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;  // might cause access violation exception
      // code ...
   }
   __except(EXCEPTION_EXECUTE_HANDLER) // warning
   {
      puts("in except");
      // code ...
   }
}
```

 Chcete-li toto upozornění opravit, použijte `GetExceptionCode` ke kontrole konkrétní výjimky před jejím zpracováním, jak je znázorněno v následujícím kódu:

```
#include <stdio.h>
#include <windows.h>
#include <excpt.h>

void f(int *p)
{
   __try
   {
      puts("in try");
      *p = 13;    // might cause access violation exception
      // code ...
   }
   __except(GetExceptionCode()==EXCEPTION_ACCESS_VIOLATION ?
               EXCEPTION_EXECUTE_HANDLER : EXCEPTION_CONTINUE_SEARCH)
   {
      puts("in except");
      // code ...
   }
}
```
