# <a name="c26105"></a>C26105
Upozornění C26105: porušení pořadí uzamknutí. Získání zámku \<lock > s úrovní \<level > způsobí inverze objednávky.

 Concurrency SAL podporuje *úrovně zámků*. Chcete-li deklarovat úroveň zámku, která je označena řetězcovým literálem bez dvojitých uvozovek, použijte `_Create_lock_level_`. Pomocí anotace `_Set_lock_level_order_(A,B)` můžete stanovit pořadí načítání mezi dvěma úrovněmi zámků, které uvádí, že zámky, které mají `A`, se musí získat před zámky, které mají úroveň `B`. K navázání hierarchie pořadí zámků (částečné řazení mezi úrovněmi zámku) použijte více anotací `_Set_lock_level_order_`. Chcete-li přidružit zámek k úrovni zámku, použijte při deklaraci zámku anotaci `_Set_lock_level_`. Upozornění C26105 se vydá, když se zjistí porušení pořadí zámků.

## <a name="example"></a>Příklad
 V následujícím příkladu je vygenerováno upozornění C26105, protože došlo k inverze pořadí zámků ve funkci `OrderInversion`.

```

_Create_lock_level_(MutexLockLevel);
_Create_lock_level_(TunnelLockLevel);
_Create_lock_level_(ChannelLockLevel);
_Lock_level_order_(MutexLockLevel, TunnelLockLevel);
_Lock_level_order_(TunnelLockLevel, ChannelLockLevel);
_Has_lock_level_(MutexLockLevel) HANDLE gMutex;

struct Tunnel
{
    _Has_lock_level_(TunnelLockLevel) CRITICAL_SECTION cs;
};

struct Channel
{
    _Has_lock_level_(ChannelLockLevel) CRITICAL_SECTION cs;
};

void OrderInversion(Channel* pChannel, Tunnel* pTunnel)
{
    EnterCriticalSection(&pChannel->cs);
    // Warning C26105
    WaitForSingleObject(gMutex, INFINITE);
    EnterCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pTunnel->cs);
    LeaveCriticalSection(&pChannel->cs);
}
```
