# <a name="c6216"></a>C6216
upozornění C6216: přetypování provedené kompilátorem mezi sémanticky rozdílnými integrálními typy: logický typ na hodnotu HRESULT

 Toto upozornění indikuje, že logická hodnota se používá jako `HRESULT` bez explicitního přetypování. Logické typy označují úspěch nenulovou hodnotou; úspěch (`S_OK`) v `HRESULT` je označen hodnotou 0. Typická hodnota selhání pro funkce, které vrací logickou hodnotu false, je stav úspěch, když je testována jako `HRESULT`. To může vést k nesprávným výsledkům.

## <a name="example"></a>Příklad
 Následující kód generuje toto upozornění:

```cpp
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  // code ...
  return IsEqual(riid1, riid2);
}
```

 Pro opravu tohoto upozornění použijte následující kód:

```cpp
#include <windows.h>
BOOL IsEqual(REFGUID, REFGUID);

HRESULT f( REFGUID riid1, REFGUID riid2 )
{
  if (IsEqual(riid1, riid2) == TRUE)
  {
    // code ...
    return S_OK;
  }
  else
  {
    // code ...
    return E_FAIL;
  }
}
```

 Pro toto upozornění je typ `SCODE` ekvivalentní k `HRESULT`.

 Další informace najdete v tématu [úspěšné makro](/windows/win32/api/winerror/nf-winerror-succeeded) a [makro, které selhalo](/windows/win32/api/winerror/nf-winerror-failed).
