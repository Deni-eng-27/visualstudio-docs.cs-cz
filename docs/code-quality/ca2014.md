# <a name="ca2014-do-not-use-stackalloc-in-loops"></a>CA2014: Nepoužívat stackalloc ve smyčkách

|Položka|Hodnota|
|-|-|
|CheckId|CA2014|
|Kategorie|Microsoft.Reliability|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Použití výrazu C# [stackalloc](/dotnet/csharp/language-reference/operators/stackalloc) uvnitř smyčky.

## <a name="rule-description"></a>Popis pravidla

Výraz jazyka C# `stackalloc` přiděluje paměť z aktuálního bloku zásobníku a Tato paměť nesmí být uvolněna, dokud aktuální volání metody nevrátí. Pokud `stackalloc` se ve smyčce používá, může to vést k přetečení zásobníku z důvodu vyčerpání paměti zásobníku.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Přesuňte `stackalloc` výraz mimo všechny smyčky v metodě.

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Může být bezpečné potlačit upozornění, když je obsažená smyčka nebo cykly vyvolána pouze v konečném počtu, což znamená, že celkové množství paměti přidělené napříč všemi `stackalloc` operacemi je poměrně malé.
