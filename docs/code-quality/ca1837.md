# <a name="ca1837-use-environmentprocessid-instead-of-processgetcurrentprocessid"></a>CA1837: Namísto Process.GetCurrentProcess().Id použijte Environment.ProcessId

|Položka|Hodnota|
|-|-|
|RuleId|CA1837
|Kategorie|Microsoft. Performance|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Toto pravidlo vyhledá volání `System.Diagnostics.Process.GetCurrentProcess().Id` a navrhne použití `System.Environment.ProcessId` místo toho, protože je efektivnější.

## <a name="rule-description"></a>Popis pravidla

`System.Diagnostics.Process.GetCurrentProcess().Id` je nákladné:

- Přidělí <xref:System.Diagnostics.Process> instanci, obvykle pouze k získání `Id` .
- <xref:System.Diagnostics.Process>Instance musí být uvolněna, což má dopad na výkon.
- Je snadné se zapomenout <xref:System.Diagnostics.Process.Dispose> na <xref:System.Diagnostics.Process> instanci.
- Pokud se kromě `Id` `Process` toho instance nepoužívá, pak se velikost propojené velikosti zvětšuje tak, že se zvětší graf typů, na které se odkazuje.
- Toto rozhraní API je trochu obtížné zjistit nebo najít.

`System.Environment.ProcessId` Vyhněte se všem výše uvedeným.

> [!NOTE]
> CA1837 pravidla je k dispozici od .NET 5,0.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Porušení lze buď opravit ručně, nebo v některých případech pomocí rychlých akcí pro opravu kódu v aplikaci Visual Studio.

##### <a name="examples"></a>Příklady

Následující dva fragmenty kódu ukazují porušení pravidla v jazyce C# a jejich opravu:

```csharp
using System.Diagnostics;

class MyClass
{
    void MyMethod()
    {
        int pid = Process.GetCurrentProcess().Id;
    }
}
```

```csharp
using System.Diagnostics;

class MyClass
{
    void MyMethod()
    {
        int pid = System.Environment.ProcessId;
    }
}
```

Následující dva fragmenty kódu ukazují porušení pravidla v Visual Basic a jejich opravu:

```vb
Imports System.Diagnostics

Class MyClass
    Private Sub MyMethod()
        Dim pid As Integer = Process.GetCurrentProcess().Id
    End Function
End Class
```

```vb
Imports System.Diagnostics

Class MyClass
    Private Sub MyMethod()
        Dim pid As Integer = System.Environment.ProcessId
    End Function
End Class
```

> [!TIP]
> Pro toto pravidlo v aplikaci Visual Studio je k dispozici oprava kódu. Pokud ho chcete použít, umístěte kurzor na porušení a stiskněte **CTRL** + **.** (tečka). **Místo příkazu Process. GetCurrentProcess () vyberte použít Environment. ProcessID. ID** ze seznamu zobrazených možností.
>
> ![Oprava kódu pro CA1837-použití "Environment. ProcessId" místo "Process. GetCurrentProcess (). Účet](media/ca1837-codefix.png)

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Porušení tohoto pravidla je bezpečné potlačit, pokud nemáte obavy z dopadu na výkon z nepotřebného přidělení a případného odstranění <xref:System.Diagnostics.Process> instance.
