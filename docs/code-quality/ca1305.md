# <a name="ca1305-specify-iformatprovider"></a>CA1305: Zadejte možnosti IFormatProvider

|||
|-|-|
|TypeName|SpecifyIFormatProvider|
|CheckId|CA1305|
|Kategorie|Microsoft.Globalizace|
|Narušující změny|Nenarušující|

## <a name="cause"></a>Příčina

Metoda nebo konstruktor volá jeden nebo více členů, <xref:System.IFormatProvider?displayProperty=fullName> které mají přetížení, které přijímají parametr a <xref:System.IFormatProvider> metoda nebo konstruktor nevolá přetížení, které přebírá parametr.

Toto pravidlo ignoruje volání metod .NET, které <xref:System.IFormatProvider> jsou dokumentovány jako ignorování parametru. Pravidlo také ignoruje následující metody:

- <xref:System.Activator.CreateInstance%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetObject%2A?displayProperty=nameWithType>
- <xref:System.Resources.ResourceManager.GetString%2A?displayProperty=nameWithType>
- <xref:System.Boolean.ToString%2A?displayProperty=nameWithType>
- <xref:System.Char.ToString%2A?displayProperty=nameWithType>
- <xref:System.Guid.ToString%2A?displayProperty=nameWithType>

## <a name="rule-description"></a>Popis pravidla

Pokud <xref:System.Globalization.CultureInfo?displayProperty=nameWithType> není <xref:System.IFormatProvider> zadán nebo objekt, výchozí hodnota, která je zadána přetíženého člena nemusí mít požadovaný účinek ve všech národních prostředích. Členové rozhraní .NET také zvolit výchozí jazykovou verzi a formátování na základě předpokladů, které nemusí být správné pro váš kód. Chcete-li se ujistit, že kód funguje podle očekávání pro vaše scénáře, měli byste zadat informace specifické pro jazykovou verzi podle následujících pokynů:

- Pokud se hodnota zobrazí uživateli, použijte aktuální jazykovou verzi. Viz třída <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>.

- Pokud hodnota bude uložena a přístup k softwaru (trvalé do souboru nebo databáze), použijte invariantní jazykovou verzi. Viz třída <xref:System.Globalization.CultureInfo.InvariantCulture%2A?displayProperty=nameWithType>.

- Pokud neznáte cíl hodnoty, zadejte jazykovou verzi spotřebitele dat nebo zprostředkovatele.

I v případě, že výchozí chování přetížené člen je vhodné pro vaše potřeby, je lepší explicitně volat přetížení specifické pro jazykovou verzi tak, aby váš kód je samodokumentující a snadněji udržované.

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Chcete-li opravit porušení tohoto pravidla, použijte <xref:System.IFormatProvider> přetížení, které trvá argument. Nebo použijte [c# interpolovaný řetězec](/dotnet/csharp/tutorials/string-interpolation) a předat <xref:System.FormattableString.Invariant%2A?displayProperty=nameWithType> metodě.

## <a name="when-to-suppress-warnings"></a>Kdy potlačit varování

Je bezpečné potlačit upozornění z tohoto pravidla, pokud je jisté, že výchozí formát je správná volba a kde udržovatelnost kódu není důležitou prioritou vývoje.

## <a name="example"></a>Příklad

V následujícím kódu `example1` řetězec porušuje pravidlo CA1305. Řetězec `example2` splňuje pravidlo CA1305 <xref:System.Globalization.CultureInfo.CurrentCulture%2A?displayProperty=nameWithType>předáním , <xref:System.IFormatProvider>který <xref:System.String.Format(System.IFormatProvider,System.String,System.Object)?displayProperty=nameWithType>implementuje , do . Řetězec `example3` splňuje pravidlo CA1305 předáním interpolovaného <xref:System.FormattableString.Invariant%2A?displayProperty=fullName]>řetězce na .

```csharp
string name = "Georgette";

// Violates CA1305
string example1 = String.Format("Hello {0}", name);

// Satisfies CA1305
string example2 = String.Format(CultureInfo.CurrentCulture, "Hello {0}", name);

// Satisfies CA1305
string example3 = FormattableString.Invariant($"Hello {name}");
```
