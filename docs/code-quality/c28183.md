# <a name="c28183"></a>C28183
Upozornění C28183: Argumentem může být jedna hodnota a je kopie hodnoty nalezené v ukazateli.

 Toto upozornění znamená, že tato hodnota je v aktuálním kontextu neočekávaná. Toto upozornění se obvykle zobrazuje, když je hodnota @no__t 0 předána jako argument funkci, která ji nepovoluje. Hodnota se ve skutečnosti našla v zadané proměnné a argument je kopií této proměnné.

 Nástroj Analýza kódu hlásí toto upozornění v prvním místě, kde může definitivně určit, že je ukazatel `NULL` nebo že obsahuje neplatnou hodnotu. Často se ale jedná o případ, kdy by k chybě mohlo dojít dříve v trasování. V takovém případě nástroj analýza kódu poskytne také číslo řádku první možné instance – obvykle v místě, kde nebylo možné s konečnou odezvou určit, zda bylo upozornění vhodné. V těchto případech je k této zprávě připojeno předchozí umístění, kde se tato situace může vyskytovat. Obvykle by se změna kódu měla objevit na čísle řádku nebo před ním, nikoli v bodě sestavy.

## <a name="example"></a>Příklad
 V následujícím příkladu Nástroj pro analýzu kódu zjistí, že *s* je v příkazu `if` null, a text `if` je pořízen. Ukazatel *s* je pak přiřazen k *t* a pak *t* je použit způsobem, kde by mohlo dojít k dereference `NULL`.

```
#include <windows.h>

int fun2(char *s)
{
    char *t;
    if (s == NULL) {
       //... but s is unchanged
    }

    t = s;

    return lstrlenA(t);
}
```

 **d: \ sample\testfile.cpp (38): Warning C28183: t ' ' by mohl být ' 0 ' a je kopie hodnoty nalezené v ' ': to nedodržuje specifikace pro funkci ' lstrlenA '.: Přím 31, 32, 36, 38**
