# <a name="c26116"></a>C26116
Upozornění C26116: selhání získání nebo udržení zámku > \<lock v \<func >.

 V jazyce není prováděno vynucování dvojic s syntakticky vymezenými zámkyC++ *a jejich zámky* pro *verze* v C/programu. Funkce může zavést vedlejší efekt uzamčení tím, že provede pozorovatelnou úpravu stavu souběžnosti. Například funkce uzamknout obálka zvyšuje počet pořizování zámků nebo počet zámků pro daný zámek. Pomocí `_Acquires_lock_` nebo `_Requires_lock_held` můžete opatřit poznámkami funkci, která má vedlejší efekt z zámku získat nebo uzamknout. Bez takovýchto poznámek se očekává, že funkce nemění počet zámků poté, co se vrátí. Pokud získání a vydání nejsou vyvážené, považují se za *osamocené*. Upozornění C26116 se vydá, když je funkce opatřena poznámkou `_Acquires_lock_`, ale nezíská zámek, nebo když je funkce opatřena `_Requires_lock_held` a uvolní zámek.

## <a name="example"></a>Příklad
 Následující příklad generuje upozornění C26116, protože funkce `DoesNotLock` byla opatřena poznámkou s `_Acquires_lock_`, ale nezíská ji. Funkce `DoesNotHoldLock` vygeneruje upozornění, protože je označeno pomocí `_Requires_lock_held` a neblokuje ji.

```

typedef struct _DATA
{
    CRITICAL_SECTION cs;
} DATA;

_Acquires_lock_(p->cs) void DoesLock(DATA* p)
{
    EnterCriticalSection(&p->cs); // OK
}

_Acquires_lock_(p->cs) void DoesNotLock(DATA* p)
{
    // Warning C26116
}

_Requires_lock_held_(p->cs) void DoesNotHoldLock(DATA* p)
{
    LeaveCriticalSection(&p->cs); // Warning C26116
}
```
