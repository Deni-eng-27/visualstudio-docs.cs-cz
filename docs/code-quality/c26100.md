# <a name="c26100"></a>C26100
Upozornění C26100: Spor. Proměnná \<var > by měla být chráněná pomocí > Lock \<lock.

 Anotace `_Guarded_by_` v kódu určuje zámek, který se použije k ochraně sdílené proměnné. Při porušení smlouvy Guard se vygeneruje upozornění C26100.

## <a name="example"></a>Příklad
 Následující příklad generuje upozornění C26100, protože došlo k porušení smlouvy `_Guarded_by_`.

```
CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA {
   _Guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA;

void Safe(DATA* p) {
   EnterCriticalSection(&p->cs);
   p->data = 1; // OK
   LeaveCriticalSection(&p->cs);
   EnterCriticalSection(&gCS);
   gData = 1; // OK
   LeaveCriticalSection(&gCS);
}

void Unsafe(DATA* p) {
   EnterCriticalSection(&p->cs);
   gData = 1; // Warning C26100 (wrong lock)
   LeaveCriticalSection(&p->cs);
}
```

 K porušení smlouvy dojde, protože ve funkci je použit nesprávný zámek `Unsafe`. V takovém případě je `gCS` správného zámku, který se má použít.

## <a name="example"></a>Příklad
 V některých případech musí být sdílená proměnná chráněná jenom pro přístup pro zápis, ale ne pro přístup pro čtení. V takovém případě použijte anotaci `_Write_guarded_by_`, jak je znázorněno v následujícím příkladu.

```

CRITICAL_SECTION gCS;

_Guarded_by_(gCS) int gData;

typedef struct _DATA2 {
   _Write_guarded_by_(cs) int data;
   CRITICAL_SECTION cs;
} DATA2;

int Safe2(DATA2* p) {
   // OK: read does not have to be guarded
   int result = p->data;
   return result;
}

void Unsafe2(DATA2* p) {
   EnterCriticalSection(&gCS);
   // Warning C26100 (write has to be guarded by p->cs)
   p->data = 1;
   LeaveCriticalSection(&gCS);
}
```

 Tento příklad také generuje upozornění C26100, protože používá nesprávný zámek ve funkci `Unsafe2`.
