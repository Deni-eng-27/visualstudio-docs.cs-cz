# <a name="c6283"></a>C6283
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Warning C6283: > \<proměnné se přidělí pomocí Array New [], ale odstraní se pomocí skalárního odstranění.  
  
 Toto upozornění se zobrazí pouze C++ v kódu a označuje, že volání funkce má nekonzistentně přidělenou paměť s operátorem Array `new []`, ale uvolní ji pomocí skalárního operátoru `delete`. Tato vada může způsobit nevracení, poškození paměti a v situacích, kdy byly operátory přepsány, dojde k chybě. Pokud je paměť přidělena pomocí pole `new []`, měla by být obvykle uvolněna pomocí `delete[]`pole.  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění:  
  
```  
void f( )  
{  
  char *str = new char[50];  
  // code ...  
  delete str;  
}  
```  
  
 Chcete-li toto upozornění opravit, použijte následující kód:  
  
```  
void f( )  
{  
  char *str = new char[50];  
  // code ...  
  delete[] str;  
}  
```  
  
 Upozornění C6283 platí pouze pro pole primitivních typů, například celá čísla nebo znaky. Pokud jsou prvky pole objekty typu třídy, je vydána výstraha [C6278](../code-quality/c6278.md) .  
  
 Použití New a DELETE má mnoho nástrah z pohledu nevracení paměti a výjimek. Chcete-li se vyhnout těmto typům nevrácených chyb a potížím s výjimkami, použijte C++ mechanismy, které jsou k dispozici pomocí knihovny Standard Template Library (STL). Mezi ně patří [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)a [Vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Další informace najdete v tématu [inteligentní ukazatele](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) a [ C++ standardní knihovna](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
