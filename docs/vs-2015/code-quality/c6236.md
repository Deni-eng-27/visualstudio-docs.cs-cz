# <a name="c6236"></a>C6236
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6236: ( \<expression> &#124;&#124; \<non-zero constant> ) je vždycky nenulová konstanta.  
  
 Toto upozornění znamená, že na pravé straně logického nebo operace, která se vyskytuje v kontextu testu, byla zjištěna nenulová konstantní hodnota, která je jiná než jedna. Logicky to znamená, že test je redundantní a lze jej bezpečně odebrat. Alternativně navrhne, aby programátor mohl použít jiný operátor, například operátor rovnosti ( `==` ), bitový operátor and () `&` nebo BITOVÝ operátor XOR ( `^` ), k otestování konkrétní hodnoty nebo příznaku.  
  
 Toto upozornění se negeneruje pro Common idiom, pokud je nenulová konstanta 1, protože je k dispozici pro selektivní povolení cest kódu v době kompilace. Nicméně upozornění je vygenerováno, pokud je nenulová konstanta vytvořena výrazem, který je vyhodnocen jako 1, například 1 + 0.  
  
## <a name="example"></a>Příklad  
 Tento kód ukazuje, jak se může zobrazit upozornění C6236. Protože není `INPUT_TYPE` 0, je výraz `n || INPUT_TYPE` vždy nenulový a `else` klauzule není nikdy provedena. Je však `INPUT_TYPE` konstanta s jinou hodnotou než jednou, což naznačuje, že je určena jako hodnota pro porovnání:  
  
```  
#define INPUT_TYPE 2  
#include <stdio.h>  
  
void f( int n )  
{  
   if ( n || INPUT_TYPE ) // analysis warning C6236  
   {  
      puts( "Always gets here" );  
   }  
   else  
   {  
      puts( "Never enters here" );  
   }  
}  
```  
  
 Následující kód místo toho používá operátor bitového `&` operátoru and () k otestování, zda `INPUT_TYPE` je nastaven bit vstupního parametru `n` :  
  
```  
#define INPUT_TYPE 2  
#include <stdio.h>  
  
void f( int n )  
{  
   if ( n & INPUT_TYPE ) // no warning  
   {  
      puts( "Bitwise-AND comparison is true" );  
   }  
   else  
   {  
      puts( "Bitwise-AND comparison is false" );  
   }  
}  
```
