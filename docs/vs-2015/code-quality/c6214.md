# <a name="c6214"></a>C6214
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
Upozornění C6214: přetypování mezi sémanticky odlišnými typy celých čísel: HRESULT na typ Boolean

Toto upozornění indikuje, že `HRESULT` se přetypování na typ Boolean. Hodnota úspěchu ( `S_OK` ) se `HRESULT` rovná 0. 0 však označuje selhání pro typ Boolean. Přetypování `HRESULT` na typ Boolean a jeho použití v testovacím výrazu vrátí nesprávný výsledek. Někdy k tomuto upozornění dochází, pokud `HRESULT` je uložen v proměnné typu Boolean. Jakékoli porovnání, které používá logickou proměnnou k otestování `HRESULT` úspěchu nebo neúspěchu, může vést k nesprávným výsledkům.

## <a name="example"></a>Příklad

Následující kód vygeneruje toto upozornění:

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);
  if ((BOOL)hr) // warning 6214
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
#include <windows.h>

BOOL f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (SUCCEEDED(hr))
  {
    // success code ...
    return TRUE;
  }
  else
  {
    // failure code ...
    return FALSE;
  }
}
```

Pro toto upozornění je `SCODE` typ ekvivalentní k `HRESULT` .

Obvykle `SUCCEEDED` `FAILED` by makro nebo mělo být použito k otestování hodnoty `HRESULT` .
