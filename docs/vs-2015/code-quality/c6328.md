# <a name="c6328"></a>C6328
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6328: Neshoda velikosti: \<type> byl předán jako parametr, \<number> Pokud \<type> je vyžadován při volání metody \<function>  
  
 Pro rutiny založené na znacích modulu C Runtime v názvu rodiny `is` *xxx* `()` , předání argumentu typu `char` může mít nepředvídatelné výsledky. Například znak jednobajtové hodnoty SBCS nebo MBCS typu `char` s hodnotou větší než `0x7F` je záporná hodnota. Pokud `char` je předán, kompilátor může převést hodnotu na signed `int` nebo signed `long` . Tuto hodnotu může kompilátor rozšířit a s neočekávanými výsledky. Například `isspace` akceptuje argument typu, `int` ale platný rozsah hodnot pro jeho vstupní argument je:  
  
 `0 <= c <= 255`a navíc se jedná o speciální hodnotu `EOF` .  
  
## <a name="example"></a>Příklad  
 Ve výchozím nastavení `char` je v Visual C++ podepsaný typ, takže rozsah hodnot proměnné typu char je `-128 <= c <= 127` . Proto pokud jste provedli následující:  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( c );  
    // code ...  
}  
  
```  
  
 `c` by se rozšířilo na znaménko `int` s hodnotou-37, která je mimo platný rozsah pro [Space](https://msdn.microsoft.com/library/3ac254c4-92aa-4238-882a-3a3fb576cdf5).  
  
 Chcete-li tento problém vyřešit, můžete použít `static_cast` , jak je znázorněno v následujícím kódu:  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( static_cast<unsigned char> (c) );  
    // code ...  
}  
  
```  
  
 K zachycení této chyby existuje upozornění C6328. U znaků v 7 bitů rozsahu ASCII je přetypování zbytečné, ale znaky mimo tento rozsah mohou mít nepředvídatelné výsledky, až do a včetně chyb a ukončení programu.
