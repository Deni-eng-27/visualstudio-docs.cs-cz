# <a name="c6328"></a>C6328
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6328: Neshoda velikosti: typ \<> předaný jako parametr \<číslo >, pokud je při volání \<funkce vyžadován > typ \<  
  
 Pro rutiny založené na znacích modulu C Runtime v názvu rodiny `is`*xxx*`()`, předání argumentu typu `char` může mít nepředvídatelné výsledky. Například znak jednobajtové znakové sady SBCS nebo MBCS typu `char` s hodnotou větší než `0x7F` je záporná hodnota. Pokud je předán `char`, kompilátor může převést hodnotu na podepsaný `int` nebo na podepsaný `long`. Tuto hodnotu může kompilátor rozšířit a s neočekávanými výsledky. Například `isspace` přijímá argument typu `int`; platný rozsah hodnot pro vstupní argument je však následující:  
  
 `0 <= c <= 255`a navíc `EOF`zvláštní hodnota.  
  
## <a name="example"></a>Příklad  
 Ve výchozím nastavení `char` je podepsaný typ ve vizuálu C++, takže rozsah hodnot proměnné typu char je `-128 <= c <= 127`. Proto pokud jste provedli následující:  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( c );  
    // code ...  
}  
  
```  
  
 `c` by se přihlásilo k podepsaným `int` s hodnotou-37, která je mimo platný rozsah pro [místo](https://msdn.microsoft.com/library/3ac254c4-92aa-4238-882a-3a3fb576cdf5).  
  
 Chcete-li tento problém vyřešit, můžete použít `static_cast`, jak je znázorněno v následujícím kódu:  
  
```cpp  
#include <iostream>  
  
void f( )  
{  
    char c = -37;  
    int retVal = isspace( static_cast<unsigned char> (c) );  
    // code ...  
}  
  
```  
  
 K zachycení této chyby existuje upozornění C6328. U znaků v 7 bitů rozsahu ASCII je přetypování zbytečné, ale znaky mimo tento rozsah mohou mít nepředvídatelné výsledky, až do a včetně chyb a ukončení programu.
