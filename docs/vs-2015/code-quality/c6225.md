# <a name="c6225"></a>C6225
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
Warning C6225: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: přiřazení 1 nebo TRUE k HRESULT. Místo toho zvažte použití S_FALSE

Toto upozornění indikuje, že `HRESULT` se přiřazuje nebo inicializuje hodnota explicitní 1. Logické typy označují úspěch nenulovou hodnotou; úspěch ( `S_OK` ) v `HRESULT` je označen hodnotou 0. Toto upozornění je často způsobeno náhodnou záměnou logických a `HRESULT` typů. K indikaci úspěchu `S_OK` by měla být použita symbolické konstanta.

## <a name="example"></a>Příklad

V následujícím kódu přiřazení `HRESULT` generuje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    // code ...
    hr = S_OK;
  }
  else
  {
    // code ...
    hr = 1;
  }
}
```

Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (SUCCEEDED(CoGetMalloc(1, &pMalloc)))
  {
    hr = S_OK;
    // code ...
  }
  else
  {
    hr = S_FALSE;
    // code ...
  }
}
```

Pro toto upozornění je `SCODE` typ ekvivalentní k `HRESULT` .

Pro indikaci selhání, `E_FAIL` nebo jiné konstanty by se měla použít.
