# <a name="c6331"></a>C6331
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6331: neplatný parametr: předávání MEM_RELEASE a MEM_DECOMMIT ve spojení s \<function> není povoleno. Výsledkem je selhání tohoto volání.  
  
 Tato zpráva znamená, že neplatný parametr předávaný do VirtualFree nebo VirtualFreeEx. VirtualFree a VirtualFreeEx obě odmítnou příznaky (MEM_RELEASE &#124; MEM_DECOMMIT) v kombinaci. Proto hodnoty MEM_DECOMMIT a MEM_RELEASE nesmí být používány společně ve stejném volání.  
  
 Nepožaduje se pro zrušení potvrzení a vydání jako nezávislé kroky. Uvolnění potvrzené paměti zruší i stránky. Také se ujistěte, že návratová hodnota této funkce není ignorována.  
  
## <a name="example"></a>Příklad  
 Následující vzorový kód vygeneruje toto upozornění:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID fd( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase,              
                0,  
                MEM_DECOMMIT | MEM_RELEASE); // warning   
  // code...  
}  
```  
  
 Chcete-li toto upozornění opravit, nepředávejte MEM_DECOMMIT VirtualFree volání, jak je znázorněno v následujícím kódu:  
  
```  
#include <windows.h>  
#define PAGELIMIT 80  
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( VOID )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  BOOL bSuccess;             
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo( &sSysInfo );    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc (  
                       NULL,                 // system selects address  
                       PAGELIMIT*dwPageSize, // size of allocation  
                       MEM_RESERVE,          
                       PAGE_NOACCESS );       
  if (lpvBase)  
  {  
    // code to access memory   
  }  
  else  
  {  
    return;  
  }  
  bSuccess = VirtualFree(lpvBase, 0, MEM_RELEASE);   
  // code...  
}  
```  
  
 Všimněte si, že použití nástrah a Free (a souvisejících rozhraní API pro přidělování dynamické paměti) má mnoho z pohledu nevracení paměti a výjimek. Chcete-li se vyhnout těmto typům nevracení a potížím s výjimkami, použijte mechanismy, které jsou k dispozici v knihovně C++ Standard Template Library (STL). Mezi ně patří [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)a [Vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Další informace naleznete v tématu [inteligentní ukazatele](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) a [Standardní knihovna C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).
