# <a name="c6211"></a>C6211
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Warning C6211: nevrácení paměti \<ukazatelem > z důvodu výjimky. Zvažte použití místního bloku catch k vyčištění paměti  
  
 Toto upozornění indikuje, že přidělená paměť není uvolněna, pokud je vyvolána výjimka. Příkaz na konci cesty může vyvolat výjimku. Analyzátor kontroluje tuto podmínku pouze v případě, že je zadána anotace `_Analysis_mode_(_Analysis_local_leak_checks_)` SAL. Ve výchozím nastavení je tato poznámka určena pro kód režimu jádra systému Windows (ovladač). Další informace o poznámkách SAL najdete v tématu [Použití poznámek SAL ke sníženíC++ vad C/kódu](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění, protože by mohla být vyvolána výjimka během druhého přidělení, čímž se následně neúnika první alokace, nebo může být výjimka vyvolána někde v kódu, který je reprezentován komentářem "`code ...`", a tím se tím nevrací jak alokace.  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4   
#include <sal.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
void f( )  
{  
    char *p1 = new char[10];  
    char *p2 = new char[10];  
  
    // code ...  
  
    delete[] p2;  
    delete[] p1;  
}  
  
```  
  
 Chcete-li použít stejné metody přidělení a odstranit tento problém, přidejte obslužnou rutinu výjimky:  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <new>  
#include <iostream>  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f()  
{  
    char *p1 = nullptr;   
    char *p2 = nullptr;  
  
    try  
    {  
        p1 = new char[10];  
        p2 = new char[10];  
  
        // code ...  
  
        delete [] p2;  
        delete [] p1;  
    }  
    catch (const bad_alloc& ba)  
    {  
        cout << ba.what() << endl;  
        delete [] p2;  
        delete [] p1;  
    }  
    // code ...  
}  
  
```  
  
 Aby nedocházelo k těmto druhům potenciálních úniků, použijte mechanismy, které jsou k C++ dispozici v knihovně STL (Standard Template Library). Mezi ně patří [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)a [Vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Další informace najdete v tématu [inteligentní ukazatele](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) a [ C++ standardní knihovna](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <vector>  
#include <memory>  
  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f( )  
{  
    // use 10-element vectors in place of char[10]  
    vector<char> v1;  
    vector<char> v2;  
  
    for (int i=0; i<10; ++i) {  
        v1.push_back('a');  
        v2.push_back('b');  
    }  
    // code ...  
  
    // use unique_ptr if you still want char[10]  
    unique_ptr<char[]> a1(new char[10]);  
    unique_ptr<char[]> a2(new char[10]);  
  
    // code ...  
  
    // No need for delete; vector and unique_ptr   
    // clean up when out of scope.  
}  
  
```
