# <a name="c6219"></a>C6219
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
Warning C6219: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: porovnává HRESULT na 1 nebo TRUE. Zvažte místo toho použití makra [Succeeded](/windows/desktop/api/winerror/nf-winerror-succeeded) nebo [Failed](/windows/desktop/api/winerror/nf-winerror-failed) .

Toto upozornění znamená, že `HRESULT` je porovnána s explicitní, nenulovou `HRESULT` hodnotou jednoho (1). Toto porovnání pravděpodobně vede k nesprávným výsledkům, protože typickou hodnotou úspěšnosti `HRESULT` ( `S_OK` ) je 0. Pokud porovnáváte tuto hodnotu s typem Boolean, implicitně se převede na false.

## <a name="example"></a>Příklad

Následující kód vygeneruje toto upozornění, protože `CoGetMalloc` vrátí `HRESULT` , což je v porovnání s `TRUE` :

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (hr == TRUE)
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

Většinou je toto upozornění způsobeno kódem, který porovnává s `HRESULT` logickou hodnotou. Je lepší použít `SUCCEEDED` `FAILED` makra nebo k otestování hodnoty `HRESULT` . Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
#include <windows.h>

void f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;
  hr = CoGetMalloc(1, &pMalloc);

  if (SUCCEEDED(hr))
  {
    // success code ...
  }
  else
  {
    // failure code
  }
}
```

Pro toto upozornění `SCODE` je typ považován za `HRESULT` .

Použití `malloc` a `free` (a souvisejících rozhraní API dynamické paměti) má mnoho nástrah jako příčinu nevracení paměti a výjimek. Chcete-li se vyhnout těmto druhům úniku a potížím s výjimkami, použijte ukazatele a třídy kontejneru poskytované standardní knihovnou jazyka C++. Mezi ně patří [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)a [Vector](/cpp/standard-library/vector). Další informace naleznete v tématu [inteligentní ukazatele](/cpp/cpp/smart-pointers-modern-cpp) a [Standardní knihovna C++](/cpp/standard-library/cpp-standard-library-reference).
