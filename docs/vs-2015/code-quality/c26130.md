# <a name="c26130"></a>C26130
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C26130: chybí anotace \_ Requires_lock_held \_ ( \<lock> ) nebo \_ No_competing_thread \_ na funkci \<func> . V opačném případě se může jednat o spor. Proměnná \<var> by měla být chráněná zámkem \<lock> .  
  
 Upozornění C26130 je vystaveno, když analyzátor detekuje potenciální konflikt časování, ale odvodí, že funkce je pravděpodobně spuštěna v jednom režimu s více vlákny, například když je funkce ve fázi inicializace založená na určitých heuristickích.  
  
## <a name="example"></a>Příklad  
 V následujícím příkladu je vygenerováno upozornění C26130, protože `_Guarded_by_` člen je upravován bez zámku.  
  
```  
  
typedef struct _DATA   
{  
    CRITICAL_SECTION cs;  
    _Guarded_by_(cs) int data;  
} DATA;  
  
void Init(DATA* p)   
{  
    p->data = 0; // Warning C26130  
}  
  
```  
  
## <a name="example"></a>Příklad  
 Je-li předchozí kód zaručen pro provoz v jednom režimu s více vlákny, přihlaste funkci pomocí `_No_competing_thread_` , jak je znázorněno v následujícím příkladu.  
  
```  
  
typedef struct _DATA   
{  
    CRITICAL_SECTION cs;  
    _Guarded_by_(cs) int data;  
} DATA;  
  
_No_competing_thread_ void Init(DATA* p)   
{  
    p->data = 0; // Warning C26130 will be resolved  
}  
  
```  
  
## <a name="example"></a>Příklad  
 Alternativně můžete opatřit fragment kódu pomocí `_No_competing_thread_begin_` a `_No_competing_thread_end_` , jak je znázorněno níže.  
  
```  
  
typedef struct _DATA   
{  
    CRITICAL_SECTION cs;  
    _Guarded_by_(cs) int data;  
} DATA;  
  
void Init(DATA* p)   
{  
    _No_competing_thread_begin_  
    p->data = 0; // Warning C26130 will be resolved  
    _No_competing_thread_end_  
}  
  
```
