# <a name="c6101"></a>C6101
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6101: vrácení neinicializované paměti  
  
 Úspěšná cesta prostřednictvím funkce nenastaví pojmenovaný `_Out_` parametr. Tato zpráva je vygenerována na základě poznámek SAL, které naznačují, že funkce v otázce je vždy úspěšná. Funkce, která nevrací indikaci úspěšnosti nebo chyby, by měla nastavit všechny jeho `_Out_` parametry, protože analyzátor předpokládá, že `_Out_` parametr je neinicializovaná data před zavoláním funkce a že funkce nastaví parametr tak, aby již není inicializovaný. Pokud funkce indikuje úspěch/selhání, pak `_Out_` parametr není nutné nastavit v případě selhání a můžete zjistit a vyhnout se neinicializovanému umístění. V obou případech je cílem zabránit čtení neinicializovaného umístění. Pokud se funkce někdy nedotkne `_Out_` parametru, který se následně používá, musí být parametr inicializován před voláním funkce a musí být označen pomocí `_Inout_` Anotace, nebo explicitní `_Pre_null_` nebo v `_Pre_satisfies_()` případě potřeby. "Částečná úspěšnost" může být zpracována s `_When_` poznámkou. Další informace najdete v tématu [Použití poznámek SAL ke snížení vad kódu C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).
