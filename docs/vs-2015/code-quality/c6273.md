# <a name="c6273"></a>C6273
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění 6273 – při volání metody Integer bylo předáno celé číslo jako parametr \<number> , pokud je vyžadováno celé číslo \<function> : Pokud je předávána hodnota ukazatele, měla by být použita% p.  
  
 Toto upozornění signalizuje, že formátovací řetězec Určuje celé číslo, například specifikace a `%d` , délku nebo prioritu pro, `printf` ale ne celé číslo, jako je `float` například řetězec, nebo `struct` je předáván jako parametr. Tato vada může mít za následek nesprávný výstup.  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění, protože místo `float` funkce do má být vyžadováno celé číslo `sprintf` :  
  
```  
#include <stdio.h>  
#include <string.h>  
  
void f_defective()  
{  
  char buff[50];  
  float f=1.5;  
  
  sprintf(buff, "%d",f);  
}  
```  
  
 Následující kód používá přetypování Integer k opravě tohoto upozornění:  
  
```  
#include <stdio.h>  
#include <string.h>  
  
void f_corrected()  
{  
  char buff[50];  
  float f=1.5;  
  
  sprintf(buff,"%d",(int)f);  
}  
```  
  
 Následující kód používá bezpečnou funkci manipulace s řetězci, `sprintf_s` aby bylo možné toto upozornění opravit:  
  
```  
#include <stdio.h>  
#include <string.h>  
  
void f_safe()  
{  
  char buff[50];  
  float f=1.5;  
  
  sprintf_s(buff,50,"%d",(int)f);  
}  
```  
  
 Toto upozornění se nevztahuje na systémy Windows 9x a Windows NT verze 4, protože% p není na těchto platformách podporováno.
